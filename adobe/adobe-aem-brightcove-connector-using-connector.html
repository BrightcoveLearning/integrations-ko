Adobe-AEM-Brightcove 커넥터: 커넥터 사용

이 항목에서는 Adobe-AEM-Brightcove Connector 5.5.x를 사용하여 AEM에서 비디오를 관리하고 게시하는 방법에 대해 설명합니다.

소개

Adobe-AEM-Brightcove 커넥터를 사용하면 AEM 내에서 Brightcove Video Cloud 비디오 및 플레이어를 관리하고 AEM 페이지에 비디오를 쉽게 포함 할 수 있습니다.

이 가이드에서는 AEM 6.3을 사용합니다. 커넥터는 AEM 6.2  및  AEM 6.4에서도 작동하며 단계는 비슷합니다.

Brightcove AEM 커넥터는 다음 구성 요소로 구성됩니다.

  • 비디오 플레이어 게시
  • 비디오 콘텐츠 관리
  • Brightcove 관리

비디오 플레이어 게시

비디오 플레이어 게시를 통해 사용자는 Brightcove Player 구성 요소를 웹 페이지로 드래그 한 다음 해당 구성 요소에 대한 Brightcove 계정, 비디오 및 플레이어를 선택할 수 있습니다. 이 구성 요소는 클래식 UI 모드의 v4.2 커넥터에서만 사용할 수 있습니다. v5.1 커넥터에서 이 구성 요소는 클래식 UI 모드와 터치 UI 모드 모두에서 사용할 수 있습니다. v4.2의 또 다른 변경 사항은 레거시 스마트 플레이어 구성 요소를 더 이상 사용할 수 없다는 것입니다. 현재 HTML5 브라이트코브 플레이어 구성 요소만 사용할 수 있습니다. 레거시 SmartPlayer 구성 요소가있는 기존 웹 페이지에는 여전히 해당 구성 요소가 포함될 수 있지만 새 레거시 SmartPlayer 구성 요소는 웹 페이지에 추가 할 수 없습니다.

비디오 콘텐츠 관리

v5.x 커넥터의 비디오 콘텐츠 관리는 AEM DAM을 사용하여 브라이트코브에 비디오를 업로드하고 브라이트코브에서 메타데이터를 업데이트합니다. 이전 버전의 커넥터에서는 의 관리 페이지가 브라이트코브에 비디오를 업로드하는 데/brightcove/admin사용되었습니다.

AEM DAM 설정

브라이트코브와 함께 AEM DAM을 처음으로 사용하기 전에 브라이트코브 구성 페이지에 지정된 DAM 통합 경로 폴더 구조를 만들어야 합니다. /system/console/configMgr 기본값은 구성 페이지에서 변경하지 않는/content/dam/brightcove_assets경우입니다. 이 폴더 아래에 각 브라이트코브 계정에 대해 폴더를 만들어야 합니다. 폴더 이름은 브라이트코브 구성의 숫자 계정 ID여야 하며 폴더 제목은 브라이트코브 구성의 계정 별칭이어야 합니다.

클래식 UI에서는 에서 폴더를 만들 수/damadmin#/content/dam있습니다. Touch UI에서 폴더는 에서 만들어집니다/assets.html/content/dam . 또는데이터베이스 동기화관리 페이지의 버튼/brightcove/admin구성된 각 Brightcove 계정에 대해 AEM DAM에 필요한 폴더를 만들고 Brightcove 계정에 이미 있는 모든 비디오를 가져옵니다. Brightcove에서 일부 또는 모든 비디오를 동기화하지 않으려면 해당 비디오를 Brightcove와 태그 AEM_NO_DAM은 AEM DAM에 동기화되지 않습니다.

또한 클래식 UI에서 AEM DAM을 사용하는 경우 볼/damadmin#/content/dam때 아래와 같이 제목 필드를 활성화합니다.

비디오가 AEM DAM에 업로드 된 다음 Brightcove에 게시되면 원본 비디오 파일은 AEM DAM에서 유지됩니다. 일부 구현의 경우 "마스터"비디오를 AEM 데이터베이스에 유지하는 것이 선호되는 반면 다른 구현의 경우 AEM 데이터베이스의 크기가 너무 빨리 증가하기 때문에 선호되지 않습니다. 원본 비디오를 AEM DAM에 저장하지 않고 브라이트코브 스튜디오 인터페이스를 사용하여 비디오를 업로드하지 않으려는 경우 브라이트코브에 게시된 후 AEM DAM에서 비디오를 삭제하면 됩니다. SYNC DATABASE작업에 의해 트리거된 다음 동기화 중 축소판 이미지 자리 표시자입니다.

또한 AEM DAM을 통해 비디오를 업로드하고 브라이트코브에 게시한 다음 나중에 브라이트코브 스튜디오에서 해당 비디오의 소스 파일을 대체하면 다음 SYNC 데이터베이스 중에 AEM DAM에 새 에셋이 생성됩니다. 소스 파일을 교체할 때 브라이트코브에 새 VideoID가 생성되기 때문입니다.  그러나 Brightcove Studio에서 AEM DAM에서 게시된 비디오에 대해 메타데이터를 업데이트할 수 있으며 예상대로 AEM DAM의 기존 자산에 대한 메타데이터가 업데이트됩니다.

클래식 UI에서 새 동영상 업로드

  1. 클래식 UI에서 새 비디오를 업로드하려면 을/damadmin#/content/dam/brightcove_assets (를) 찾아 사용할 브라이트코브 계정의 폴더를 연 다음새로 만들기>새 파일을 클릭합니다.
  2. 업로드한 후 제목 필드를 편집하거나 비워 두면 에셋을 게시할 때 DAM에 업로드된 비디오 파일의 이름으로 설정되어 브라이트코브에 업로드됩니다.   제목 필드는 새 자산의 제목 필드를 클릭하여 DAM 자산 목록보기에서 편집 할 수 있습니다.
  3. 또는 새 에셋을 열고제목필드를 편집하고 변경 사항을 저장할 수 있습니다.
  4. Brightcove 사용자 정의 필드를 제외하고 다른 메타 데이터 필드는 선택 사항입니다. 그러나 AEM 클래식 UI의 제한으로 인해 커넥터의 5.x 버전에서 지원이 추가 된 Brightcove 사용자 정의 필드는 클래식 UI에서 편집 할 수 없으며 대신 Touch UI를 사용해야합니다.
  5. 이 시점에서 비디오는 AEM DAM에만 있습니다. 비디오를 Brightcove에 동기화하려면 자산 목록보기에서 새 자산을 선택하고 활성화를 클릭하여 비디오를 Brightcove에 게시합니다.

  2. 비디오를 업로드한 후 원하는 경우제목필드를 편집합니다. 제목 필드는 새 자산을 열고 다음을 선택하여 편집합니다. 속성 보기기본 속성 페이지에서 제목 편집:
  3. Brightcove 사용자 정의 필드를 제외하고 다른 메타 데이터 필드는 선택 사항입니다.
  4. 비디오를 Brightcove와 동기화하려면 자산보기 내에서 게시를 선택합니다.
  5. 또는 목록, 카드 또는 열보기에서 새 자산을 선택하고 게시를 클릭하여 비디오를 Brightcove에 게시 할 수 있습니다.

AEM DAM에서 축소판 및 포스터 이미지 변경

DAM의 비디오 자산에서 brc_thumbnail.png 및 brc_poster.png라는 변환을 덮어 쓰면 Brightcove에서 썸네일 및 포스터 이미지를 업데이트 할 수 있습니다. 클래식 UI에서 / damadmin # / content / dam / brightcove_assets로 이동하고 비디오 자산을 열고 변환 탭에서 업로드를 클릭합니다.

brc_thumbnail.png 또는 brc_poster.png 업로드 후 에셋을저장및활성화하여 새 썸네일 및 포스터 이미지를 브라이트코브에 업로드합니다. Touch UI에서 비디오 에셋을/assets.html/content/dam/brightcove_assets/찾아 엽니다. 딸깍 하는 소리변환아래 그림과 같이:

그런 다음 변환 추가를 클릭하고 brc_thumbnail.png 또는 brc_poster.png라는 파일을 업로드 한 다음 자산을 게시하여 새 썸네일 및 포스터 이미지를 Brightcove에 업로드합니다.

  AEM에서 브라이트코브로 비디오 가져오기

새 비디오가 Brightcove Studio에서 직접 업로드되는 경우 및/또는 메타데이터가 Brightcove Studio에서 직접 편집되는 경우  NS 데이터베이스 동기화 Brightcove 관리 페이지의 버튼/brightcove/admin새 비디오 또는 업데이트된 메타데이터를 Brightcove에서 AEM DAM으로 동기화하는 데 사용됩니다.  브라이트코브에서 DAM으로 가져온 비디오 에셋은 원본 비디오 파일을 AEM DAM에 복사하지 않고 대신 브라이트코브 축소판을 에셋의 자리 표시자로 사용합니다. 따라서 Brightcove에서 비디오를 가져올 때 AEM 데이터베이스 크기가 크게 증가하지 않습니다.

데이터베이스 동기화도 일정에 따라 실행되도록 설정할 수 있습니다. 에서/system/console/configMgr구성 콘솔을 연 다음 “브라이트코브 자산 통합 Cronjob 스케줄러 구성”을 찾아 엽니다.

CRON 활성화를 클릭하고 CRON 스케줄러 필드에 유효한 CRON 표현식을 입력합니다.

다음 명령을 OS에서 수동으로 사용하여 동기화 프로세스를 트리거 할 수도 있습니다.

/usr/bin/curl
---anyauth —사용자 사용자 < 이름>:  < 암호> — 자동 압축 < 인스턴스>/빈/브라이트코브/데이터 애드. HTML

참고 < 사용자 이름>: < 암호> Brightcove 계정에 대한 액세스 권한과 태그 및 콘텐츠를 만들 수 있는 권한이 있는 유효한 AEM 사용자여야 합니다.

Brightcove 관리

이전 버전의 커넥터에서는 의 브라이트코브 관리 페이지를 사용하여 브라이트코브에 비디오를 업로드하고 해당 비디오의 메타데이터를/brightcove/admin편집했습니다. AEM DAM에 대한 새로운 지원을 통해 이제 Brightcove 관리 페이지는 일반적으로 Brightcove 계정의 데이터를 보는 데만 사용됩니다. 예를 들어, Brightcove 관리 페이지는 Brightcove에 대한 직접 API 호출을 사용하여 Brightcove 계정의 비디오를 실시간으로 표시하고 새 비디오가 Brightcove에 업로드되었는지 또는 메타 데이터가 다음의 게시 프로세스에 의해 Brightcove에 동기화되었는지 확인하는 데 사용할 수 있습니다. AEM DAM.

Brightcove에서의 재생을 확인하기 위해 Brightcove 구성 페이지 (/ system / console / configMgr)에 지정된 기본 비디오 및 재생 목록 플레이어를 사용하여이 Brightcove 관리 페이지에서 비디오를 미리 볼 수 있습니다.

브라이트코브 관리 페이지에는 브라이트코브에서 AEM DAM으로의 동기화 프로세스를 시작하는 SYNC DATABASE단추가 포함되어 있습니다. SYNC DATABASE단추를 눌러 구성된 모든 브라이트코브 계정을 AEM DAM에 동기화합니다.   브라이트코브의 일부 또는 모든 비디오를 동기화하지 않으려는 경우 브라이트코브 태그 AEM_NO_DAM이 있는 동영상은 AEM DAM에 동기화되지 않습니다.

이 동기화 프로세스는 cron 작업을 통해 자동화 할 수 있으므로 수동 작업이 필요하지 않습니다.

비디오에 텍스트 트랙 추가

현재 텍스트 트랙은 의 브라이트코브 관리 페이지를 통해서만 비디오에 추가할 수/brightcove/admin있습니다. 텍스트 트랙은 새 텍스트 트랙업로드버튼을 통해 추가됩니다.

DAM에서 변환으로 업로드 된 텍스트 트랙을 지원하는 것은 나중에 구현할 로드맵에 있습니다.

포스터 이미지 업데이트

AEM DAM에서 포스터 및 썸네일 이미지를 업데이트하는 것 외에도 브라이트코브 관리 페이지를 통해 원격 HTTP URL에서 브라이트코브에 업로드할 수도/brightcove/admin있습니다.
