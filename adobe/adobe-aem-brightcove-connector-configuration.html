Adobe-AEM-Brightcove 커넥터: 구성

이 항목에서는 AEM (Adobe Experience Manager) 용 Brightcove 커넥터 구성에 대해 설명합니다.

소개

Adobe-AEM-Brightcove 커넥터를 사용하면 AEM 내에서 Brightcove Video Cloud 비디오 및 플레이어를 관리하고 AEM 페이지에 비디오를 쉽게 포함 할 수 있습니다.

이 가이드에서는 AEM 6.3을 사용합니다. 커넥터는 AEM 6.2, 6.4 및 6.5에서도 작동하며 단계는 비슷합니다.

구성 단계

다음 섹션에서는 클래식 인터페이스를 사용하여 Adobe-AEM-Brightcove 커넥터를 구성하는 단계를 안내합니다. 이 단계를 완료하기 전에 커넥터를 설치해야 합니다. Adobe-AEM-Brightcove 커넥터를 참조하십시오. 설치

계정 정보 입력

이제 Video Cloud 계정 정보로 AEM-Brightcove 커넥터를 구성 할 준비가되었습니다.

  1. AEM 웹 콘솔 구성 관리자 열기 (http://localhost:4502/system/console/configMgr)
  2. 브라이트코브 서비스에서검색하여 서비스를 찾습니다.
  3. +기호를 클릭하여 새 구성을 추가합니다.
  4. 대화 상자에서 아래와 같이 값을 입력합니다. 보다 API 인증 자격 증명 관리클라이언트 ID 및 클라이언트 암호를 가져오는 방법에 대한 지침은 여기에 필요한 권한은 다음과 같습니다.
    • 다음은 구성 양식에 대해 제안되는 값입니다.
      • 계정 별칭 : 커넥터에 표시할 계정의 이름을 입력합니다 (필수).
      • 계정 ID : 브라이트코브 게시자 ID를 입력합니다 (필수).
      • 클라이언트 ID : 브라이트코브 API 인증 페이지에서 브라이트코브 클라이언트 ID를 입력합니다 (필수).
      • 클라이언트 시크릿 : 브라이트코브 클라이언트 시크릿 API 입력 (필수)
      • 허용된 그룹 : 커넥터에 액세스할 그룹을 지정합니다 (필수) 이 커넥터가 포함된 그룹인지 확인합니다.
      • 플레이어 스토어 경로 : 기본값을 유지하거나 이 브라이트코브 계정의 플레이어에게 선택한 폴더를 사용합니다 (필수)
      • 기본 비디오 플레이어 ID : 기본값을 유지하거나 다른 플레이어 ID 선택 (필수)
      • 기본 비디오 플레이어 키 : 더 이상 사용되지 않음, 비워 두거나 이전 커넥터의 값 사용
      • 기본 재생 목록 플레이어 ID : 재생 목록을 사용하려는 경우 재생 목록에 대해 활성화된 플레이어 ID를 입력합니다 (필수).
      • 기본 재생 목록 플레이어 키 : 더 이상 사용되지 않음, 비워 두거나 이전 커넥터의 값 사용
      • 프록시 서버 : 이전에 프록시를 사용하고 있는 경우 비워 두거나 프록시를 입력합니다.
      • 댐 통합 경로 : 기본값을 유지하거나 이 브라이트코브 계정의 비디오 에셋에 대해 선택한 폴더를 사용합니다 (필수)
      • 인제스트 프로필 : 업로드된 비디오에 사용할 브라이트코브 수집 프로필을 지정하고, 빈 값은 기본 프로필을 사용합니다. 빈/기본값이 작동하지 않는 경우 Brightcove 지원팀에 비디오 클라우드 백엔드에 기본 프로필이 정의되어 있는지 물어보십시오.
  5. 구성을 테스트하십시오.  http://localhost:4502/brightcove/admin 로 이동하여 오른쪽 상단 모서리의 풀다운에서 방금 추가한 브라이트코브 계정을 선택합니다. 동영상이 표시되지 않으면 다음 이유 중 하나 때문일 수 있습니다.
    1. API 자격 증명 계정 ID , 클라이언트 ID및클라이언트 암호가 다음과 같을 수 있습니다.커넥터 구성에 잘못 입력되었습니다.
    2. API 자격 증명이 Brightcove에서 제대로 구성되지 않았습니다.
    3. 현재 로그인 한 사용자는 다음 중 하나의 구성원이 아닙니다. 허용 된 그룹 커넥터 구성
    4. 네트워크 구성이 Brightcove의 클라우드 서버에 대한 API 액세스를 차단하고 있습니다. 자세한 내용은“네트워크 및 API 연결 문제 디버깅”을참조하십시오.
  6. 여러 Video Cloud 계정을 사용하는 경우 각 계정에 대해 이전 단계를 반복하십시오.

Brightcove 플레이어 설정

마지막 구성 작업은 Brightcove 플레이어를 설정하는 것입니다.

  1. 사이트 관리 도구 (클래식 UI의 http://localhost:4502/siteadmin#/content 또는 Touch UI의 http://localhost:4502/sites.html/content) 로 이동합니다.
  2. 최상위 웹 사이트 폴더에서 구성의 플레이어 스토어 경로와 일치하는 새 폴더를 만듭니다. 기본값은 구성의 폴더 이름을  brightcovetools라고 하며, 플레이어 안에 폴더가있습니다. 여러 브라이트코브 계정에 연결하는 경우 각 계정에 대해 고유한 플레이어 폴더를 만들 수 있습니다.
  3. 플레이어폴더 내에 새 브라이트코브 플레이어 페이지가 생성됩니다.
  4. 페이지를 만든 후 페이지를 마우스 오른쪽 버튼으로 클릭하고 클래식 UI의 메뉴에서 속성을 선택합니다. Touch UI에서 페이지를 선택한 다음 속성을 클릭합니다.
  5. 대화 상자의 Brightcove 기본 탭과 Brightcove 계정으로 이동 한 다음 목록에서 플레이어를 선택합니다.
  6. 여러 플레이어를 게시 할 수 있도록하려면 위의 단계를 반복하여 각 플레이어에 대한 페이지를 만듭니다.
  7. 재생 목록도 사용하려면 위 단계를 반복하여 재생 목록 플레이어 용 페이지를 만듭니다.
  8. 여러 Brightcove 계정을 사용하는 경우 각 계정에 대한 플레이어 페이지를 만들어야합니다.

Brightcove 자산 위치에 대한 메타 데이터 스키마 활성화

v5.4 커넥터 이상에서 Touch UI를 사용하는 경우 브라이트코브 자산 위치에 대한 메타데이터 스키마를 활성화해야 합니다.  클래식 UI만 사용하는 경우에는 필요하지 않습니다. 다음 단계를 수행하여 메타데이터 스키마를 활성화합니다.

  1. AEM 도구를 엽니 다.
  2. 에셋 선택:
  3. 메타데이터 스키마 선택:
  4. 브라이트코브를 선택하고 폴더에 적용 을 클릭합니다.
  5. 브라이트코브_assets 폴더가 없으면 브라이트코브_assets 라는 폴더를 만듭니다.
  6. “브라이트코브_애셋”이라는 이름을 입력합니다.
  7. 브라이트코브_애셋을 선택하고 적용 을 클릭합니다.

이제 AEM-Brightcove 커넥터의 구성을 완료했습니다. 커넥터사용으로진행합니다.
