WordPress-Brightcove 커넥터: 자주 묻는 질문

이 항목에는 Wordpress-Brightcove 커넥터에 대해 자주 묻는 질문이 포함되어 있습니다.

공개적으로 액세스 할 수없는 WordPress 설치에서 플러그인을 실행할 수 있습니까?

예, 공개 여부에 관계없이 작동하지만 상태 업데이트, 동영상 업로드 등과 같이 사이트에 대한 콜백이 필요한 기능은 사용할 수 없습니다.

 플러그인/테마 개발자를 위한 필터가 있습니까?  

brightcove_account_actions = [편집, 삭제]; // Brightcove 소스를 조작 할 때 사용할 수있는 작업.

다중 사이트에서 작동합니까?

네, 그렇습니다.

둘 이상의 Brightcove 계정을 사용할 수 있습니까?

예, 원하는 경우 여러 Brightcove 계정의 소스를 추가 할 수 있습니다.

사용자에게 다른 권한을 설정할 수 있습니까?

예. 이 문서에서 사용자 권한 매트릭스를 찾을 수있습니다. 사용자 역할을 변경하려면 워드프레스 add_cap 함수를 사용하는 것이 좋습니다. https://codex.wordpress.org/add_cap

동기화는 어떻게 작동합니까?

플러그인은 디스플레이를 위해 Brightcove API에서 직접 정보를 가져오고 로컬로 비디오를 동기화하지 않습니다.

최대 업로드 파일 크기를 늘리려면 어떻게해야합니까?

최대 파일 크기는 웹 서버 및 PHP 구성에 따라 결정됩니다. php.ini 에서 업로드_max_filesize와 post_max_size 값을 설정해야 합니다. php_ini_loaded_file () 은 PHP.ini 위치를 찾는 데 도움이 될 수 있습니다.

비디오를 업데이트 한 사람과 업데이트 한시기를 확인할 수 있도록 변경 내용 추적을 활성화하려면 어떻게해야합니까?

내부 이름이 '_change_history'인 텍스트' 유형의 사용자 정의 필드를 만듭니다. 비디오가 업데이트 될 때마다 사용자 이름과 현재 시간이이 필드에 기록 된 변경 사항 목록에 추가됩니다.
