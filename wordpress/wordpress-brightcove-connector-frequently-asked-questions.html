페이지 내용

    워드 프레스 브라이트코브 커넥터: 자주하는 질문

    이 항목에는 워드프레스-브라이트코브 커넥터에 대한 질문과 대답이 포함되어 있습니다.

    공개적으로 액세스 할 수없는 WordPress 설치에서 플러그인을 실행할 수 있습니까?

    예, 공개 여부에 관계없이 작동하지만 상태 업데이트, 동영상 업로드 등과 같이 사이트에 콜백이 필요한 기능은 사용할 수 없습니다.

     플러그인/테마 개발자를위한 필터가 있습니까?  

    Brightcove_account_action = [편집, 삭제];//브라이트코브 소스를 조작할 때 사용할 수 있는 액션입니다.

    멀티 사이트에서 작동합니까?

    네, 그럴 것입니다.

    브라이트코브 계정을 두 개 이상 사용할 수 있습니까?

    예. 원하는 경우 여러 브라이트코브 계정의 소스를 추가할 수 있습니다.

    사용자에 대해 다른 권한을 설정할 수 있습니까?

    예. 이 문서에서사용자 권한 매트릭스를 찾을 수 있습니다. 사용자 역할을 변경하려면 WordPress add_cap 함수를 사용하는 것이 좋습니다. https://codex.wordpress.org/add_cap

    동기화는 어떻게 작동합니까?

    플러그인은 단순히 브라이트코브 API에서 직접 정보를 가져올 뿐이며 비디오를 로컬에서 동기화하지 않습니다.

    최대 업로드 파일 크기를 늘리려면 어떻게 해야 합니까?

    최대 파일 크기는 웹 서버 및 PHP 구성에 의해 결정됩니다. 당신은 당신의 php.ini 에 업로드 맥스 파일 크기 및 포스트 최대 크기의 값을 설정해야합니다. php_ini_loaded_file () 은 당신의 PHP.ini 위치에있는 위치를 찾는 데 도움이 될 수 있습니다.

    누가 비디오를 업데이트했는지 언제 업데이트했는지 확인할 수 있도록 변경 내용 추적을 활성화하려면 어떻게 해야 합니까?

    내부 이름이 '_change_history'인 '텍스트'유형의 사용자 정의 필드를 만듭니다. 동영상이 업데이트될 때마다 사용자 이름과 현재 시간이 이 필드에 기록된 변경 사항 목록에 추가됩니다.

    nginx의 경우:
    http://nginx.org/en/docs/http/ngx_http_core_module.html#client_max_body_size (클라이언트_max_body_size) 아파치의
    경우:
    (리미트 요청본문)
      http://httpd.apache.org/docs/current/mod/core.html#limitrequestbody
