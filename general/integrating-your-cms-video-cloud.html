사용자 기능

다음은 CMS 사용자에게 제공 할 수있는 Video Cloud 관련 기능입니다.

  • Video Cloud에 새 비디오 추가
  • Video Cloud 비디오를 새 버전으로 교체
  • 제목, 설명, 태그와 같은 동영상의 메타 데이터 업데이트
  • 동영상 삭제
  • 재생 목록 만들기
  • 재생 목록의 비디오 변경
  • 재생 목록 삭제
  • 비디오 플레이어 만들기
  • 크기 또는 스타일과 같은 비디오 플레이어 속성 수정
  • 플러그인을 통해 비디오 플레이어에 특수 기능 추가
  • 단일 비디오 또는 재생 목록 게시
  • 비디오로드, 조회수, 재생률, 참여 등에 대한 분석 데이터를 제공합니다.

이 모든 기능을 최종 사용자에게 공개하고 싶지 않을 수 있습니다. 예를 들어, 최종 사용자가 비디오를 삭제하지 못하게하는 것을 원하지 않을 수 있습니다. 사용자가 Video Cloud Studio로 직접 이동하지 않고 Video Cloud를 CMS와 통합 할 때의 장점 중 하나는 Brightcove API를 통해 사용자에게 노출 할 기능을 정확히 선택할 수 있다는 것입니다.

입증

모든 Brightcove API 요청에 대해 인증은 OAuth2 액세스 토큰을 기반으로합니다. 액세스 토큰을 얻기위한 두 단계 프로세스가 있습니다.

  1. 필요한 API 작업에 대한 권한이있는 클라이언트 자격 증명 만들기
  2. 클라이언트 자격 증명을 사용하여 API 요청을 인증하는 임시 액세스 토큰을 만듭니다.

클라이언트 자격 증명 생성

클라이언트 자격 증명 생성은 Video Cloud Studio또는 OAuth API를 통해 수행할 수 있는 일회성작업입니다. 그러나 이 작업을 수행하면client_id및client_secret가 반환되므로 액세스 토큰을 요청하려면 저장해야 합니다.

client_secret는 클라이언트 자격 증명이 생성된 경우에만 표시되며, 다시는 검색할 수 없습니다. 클라이언트 암호를 분실한 경우 새 클라이언트 자격 증명을 만들어야 합니다.

액세스 토큰 만들기

임시 액세스 토큰은 OAuth API를 사용하여 생성됩니다 . client_id및 는 Base64로 인코딩되고 a로client_secret전달되어야 합니다. Basic권한 부여 문자열입니다.

access_token반환된 값은 API 호출과 함께 권한 부여 헤더에 전달되었습니다.

    >Authorization: Bearer your_access_token

액세스 토큰은 5 분 동안 유효합니다. 수백 번의 연속적인 API 호출을 수행하는 일종의 일괄 작업을 수행하지 않는 한, 시간 제한을 추적하는 대신 각 API 호출에 대해 새 작업을 요청하는 것이 좋습니다.

비디오 추가

사용자가 CMS에서 Video Cloud에 비디오를 추가할 수 있도록 하려면동적 인제스트 API를 사용하여 이 작업을 수행할 수있습니다. 사용자가 S3 버킷 또는 공용 서버 일 수있는 리포지토리에 비디오를 업로드하도록하는 것이 좋습니다. 동적 인제스트 시스템은 아래에 설명된 2단계 프로세스를 통해 비디오를 가져와서 비디오 클라우드 시스템에 추가할 수 있습니다.

Video Cloud에 비디오 개체 추가

첫 번째 단계는 CMS API에POST요청하여 비디오 클라우드 시스템에서 비디오 객체를 만드는 것입니다.

    https://cms.api.brightcove.com/v1/accounts/:account_id/videos

요청 본문에는 JSON 객체의 기본 비디오 속성 (최소 비디오name속성) 이 포함되지만 다음과 같은 추가 메타데이터를 포함할 수도 있습니다. description및tags :

    {
    "name": "Woodpecker",
    "description": "A bird that hunts insects inside wood",
    "reference_id": "Bird_Woodpecker.mp4",
    "tags": ["bird", "air", "nature"]
    }
A는 모든 비디오에 추가할reference_id수 있습니다. 참조 ID는 계정 내에서 고유해야합니다. 동영상에 CMS 시스템에 ID가 있는 경우 Video Cloud 시스템에서 해당 ID를 참조 ID로 사용하는 것이 좋습니다.

비디오 수집

비디오 객체를 만들면 CMS API는 비디오 속성을 포함하는 JSON 객체를 반환합니다. id JSON에서 비디오를 추출하고 이를 사용하여동적 인제스트 API를호출하여 비디오의 수집 및 트랜스코딩을 요청합니다.

    https://ingest.api.brightcove.com/v1/accounts/ACCOUNT_ID/videos/VIDEO_ID/ingest-requests

다시 동영상 파일의 위치를 지정하는 요청 본문에 JSON을 보냅니다.

    {
      "master":{
        "url":"http://learning-services-media.brightcove.com/videos/mp4/Bird_Woodpecker.mp4"
      },
      "profile":"multi-platform-extended-static",
      "capture-images": true
    }
위의 코드는 사용자에게 수집 프로필 선택기를 제공한다고 가정합니다. 그렇지 않은 경우 요청 본문에서profile필드를 생략해야 합니다. 이 경우 계정 기본 프로필이 사용됩니다. 여러 인제스트 프로파일이 필요하지 않는 한 브라이트코브가 권장하는 것이 일반적입니다.

profile다음은 트랜스코딩 프로세스에서 만들어야 하는 변환을 지정하는 인제스트 프로필입니다. 대부분의 경우 다음 표준 프로필 중 하나가 적합해야합니다.

Dynamic Delivery 프로필

  • multi-platform-extended-static
  • multi-platform-standard-static

레거시 수집 프로필

  • videocloud-default-v1 (the default)
  • screencast-1280
  • smart-player-transition
  • single-bitrate-high
  • audio-only
  • single-bitrate-standard
  • high-resolution

그러나 필요한 경우 다음을 사용하여 추가 사용자 지정 수집 프로필을 만들 수 있습니다. 프로파일 수집 API또는 사용비디오 클라우드 스튜디오 .

비디오 관리

CMS API를사용하면 계정에 대한 비디오 데이터를 다시 가져올 수 있습니다. (위에 표시된 것처럼 비디오 수집 프로세스의 일부로 비디오 개체를 만드는데도 사용됩니다.) 가장 기본적인 요청은 다음과 같습니다.

    https://cms.api.brightcove.com/v1/accounts/account_id/videos

기본적으로이 요청은 이름, 설명, 태그, 사용자 지정 필드, 생성 및 마지막 수정 날짜, 포스터 및 미리보기 이미지의 URL 등 다양한 메타 데이터를 포함하는 20 개의 동영상 개체의 JSON 배열을 반환합니다.

다음 매개 변수 중 하나 이상을 요청에 추가하여 요청 결과를 구체화 할 수 있습니다.

limit
이것은 반환 할 비디오 개체의 수를 결정하며 100까지 임의의 수로 설정할 수 있습니다. 기본값은 20입니다.
offset
건너뛸 항목 수를 결정하므로 비디오 카탈로그를 통해 페이지 페이지와limit함께 사용됩니다. 기본값은 0입니다.
sort
결과를 정렬 할 비디오 메타 데이터 필드를 결정합니다. 기본적으로 결과는 다음을 기준으로 정렬됩니다. updated_at (내림차순, 가장 최근에 업데이트 된 동영상을 먼저 표시)

보다 CMS API개요 - 매개변수이러한 매개변수에 대한 자세한 내용은

비디오 검색

q매개 변수를 사용하여 다양한 기준으로 비디오를 검색할 수도 있습니다. 이름, 설명, 태그, 날짜 및 비디오 상태와 같은 특정 필드로 검색 할 수 있습니다.

    https://cms.api.brightcove.com/v1/accounts/account_id/videos?q=tags:sea,mammal

자세한 내용 및 검색 옵션에 대한 자세한 내용은 비디오검색을 참조하십시오 .

특정 비디오 가져 오기 및 업데이트

해당 ID 또는 참조 ID로 특정 비디오를 검색하려면

    https://cms.api.brightcove.com/v1/accounts/account_id/videos/id
    or
    https://cms.api.brightcove.com/v1/accounts/account_id/videos/ref:reference_id

GET요청은 비디오 객체를 반환합니다. 업데이트하려면 를JSON수정하고PATCH요청을 사용하여 동일한 URL에 반환합니다.

사용자에게 비디오를 표시할 때 게시하거나 업데이트할 잘못된 비디오를 선택할 가능성을 줄이기 위해 디스플레이에 축소판 이미지를 포함하는 것이 좋습니다.

재생 목록

재생 목록 정보는 비디오 정보와 거의 동일한 방식으로 CMS API를사용하여 관리됩니다. Video Cloud는 두 가지 범주에서 8가지 유형의 재생 목록을 지원합니다.

수동 (또는EXPLICIT ) 재생목록
지정된 동영상 세트 포함-최대 100 개의 동영상이 포함될 수 있습니다.
스마트 재생 목록
검색 기준에 따라 런타임에 동적으로 빌드됨 - 목록 내에서 비디오가 정렬되는 방식에 해당하는 7가지 종류의 스마트 재생 목록이 있습니다.
  • ACTIVATEDOLDESTTONEWEST
  • ACTIVATEDNEWESTTOOLDEST
  • ALPHABETICAL
  • PLAYSTOTAL
  • PLAYSTRAILINGWEEK
  • STARTDATEOLDESTTONEWEST
  • STARTDATENEWESTTO_OLDEST
할 수 있습니다. ~ 아니다재생 목록이 생성된 후 수동 재생 목록을 스마트 재생 목록으로(또는 그 반대로) 변경합니다.

동영상 개수 제한은 최대 100 개까지 설정할 수 있습니다.

동영상과 마찬가지로 계정에 재생 목록이 많은 경우limit및offset를 사용하여 모든 재생 목록을 검색할 수 있습니다.

    https://cms.api.brightcove.com/v1/accounts/account_id/playlists

반환된 재생 목록 객체 배열에는 위에서 설명한 유형 중 하나에type해당하는 를 포함하여 재생 목록의 메타데이터가 포함됩니다. 유형이 인EXPLICIT경우 포함된 비디오의 ID를 포함하는video_ids배열도 있습니다. 유형이 스마트 재생 목록 유형 중 하나인 경우 다음과 같이 비디오를 검색하는 검색 문자열이 포함된search속성이 있습니다.

    q=tags:fish,birds

다음과 같이 단일 재생 목록을 검색할 수도 있습니다. id

    https://cms.api.brightcove.com/v1/accounts/account_id/playlists/playlist_id

페이지의 동영상에 대한 정보를 표시하기 위해 재생 목록의 전체 비디오 개체를 검색해야 하는 경우 해당 URL에/videos추가하기만 하면 됩니다.

    https://cms.api.brightcove.com/v1/accounts/account_id/playlists/playlist_id/videos

스마트 재생 목록의 경우 요청은 현재 검색 기준과 일치하는 동영상을 반환하지만 변경 될 수 있습니다.

플레이어 생성

브라이트코브 플레이어는플레이어 관리 API를 통해 생성할 수있습니다. API를 사용하면 플레이어를 생성하고, 프로퍼티를 업데이트하고, 페이지에 포함할 URL, iframe태그 또는 HTML 블록의 형태로 임베드 코드를 가져올 수 있습니다.

계정 당 최대 200 명의 플레이어를 사용할 수 있지만 일반적으로 사용자가 절대적으로 필요한만큼의 플레이어를 보유하는 것이 덜 혼란 스럽습니다. 단일 비디오 또는 재생 목록을 재생하려면 별도의 플레이어가 있어야하지만 그렇지 않으면 스타일이 다르게 지정되거나 플러그인을 통해 다른 기능이 추가 될 때만 다른 플레이어가 필요합니다.

플레이어를 생성하려면 플레이어 관리 API에POST요청하기만 하면 됩니다.

    https://players.api.brightcove.com/v2/accounts/account_id/players

요청 본문에플레이어 구성을포함하십시오. 필요한 것은 다음과name같습니다.

    {
    "name": "Single video player for blog posts"
    }

응답은 플레이어 ID와 여러 형식의 소스 코드를 제공합니다.

    {
    "embed_code": "<iframe src='//players.brightcove.net/57838016001/de055fa4-4f09-45af-8531-419c6794ad04_default/index.html' allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>",
    "embed_in_page": "http://players.brightcove.net/57838016001/de055fa4-4f09-45af-8531-419c6794ad04_default/in_page.embed",
    "id": "de055fa4-4f09-45af-8531-419c6794ad04",
    "preview_embed_code": "<iframe src='//preview-players.brightcove.net/v1/accounts/57838016001/players/de055fa4-4f09-45af-8531-419c6794ad04/preview/embeds/default/master/index.html' allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>",
    "preview_url": "http://preview-players.brightcove.net/v1/accounts/57838016001/players/de055fa4-4f09-45af-8531-419c6794ad04/preview/embeds/default/master/index.html",
    "url": "http://players.brightcove.net/57838016001/de055fa4-4f09-45af-8531-419c6794ad04_default/index.html"
    }

전체 플레이어 구성을 가져오려면/players엔드포인트에 요청하지만 위의 응답에서 반환되는 플레이어 ID를 추가합니다.

    https://players.api.brightcove.com/v2/accounts/account_id/players/de055fa4-4f09-45af-8531-419c6794ad04

동일한 엔드포인트에PATCH요청하여 플레이어 구성을 업데이트할 수 있습니다.

위의 응답에서preview_embed_code및 를 확인할 수preview_url있습니다. 새 플레이어 또는 플레이어 업데이트를 테스트 할 수 있도록 새로 생성되거나 업데이트 된 플레이어가 미리보기 모드로 설정되어 기존 플레이어에 변경 사항을 적용하기 전에 확인할 수 있습니다. 변경 사항을 프로덕션으로 푸시하려면 다음을 수행해야 합니다. 출판하다이 요청이 있는 플레이어:

    https://players.api.brightcove.com/v2/accounts/account_id/players/de055fa4-4f09-45af-8531-419c6794ad04/publish

플레이어 커스터마이징

브라이트코브 플레이어는다음과 같은 표준 웹 기술로 구축되었습니다. HTML, CSS 및 JavaScript. 동일한 기술을 사용하여 플레이어를 사용자 지정할 수 있습니다. 이 작업은 플레이어가 게시된 페이지에서 수행할 수 있지만, 플레이어구성을통해 플레이어 자체에 사용자 지정을 추가하고 다음을 통해 플레이어를 업데이트하는 것이 좋습니다. PATCH이전 섹션에서 설명한 대로 플레이어 관리 API에 요청합니다.

JavaScript를 통해 플레이어에 추가 기능을 추가할 수도 있습니다. 플러그인 , 그리고 광범위한플레이어 API코드를 플레이어와 통합하는 데 도움이 됩니다. Brightcove는 광고 활성화, 최종 화면 사용자 지정 및 오버레이 추가와 같은 작업을 위해 미리 만들어진 다양한 플러그인을 제공합니다.

동영상 게시

에서플레이어 만들기 섹션위에서 우리는 플레이어 관리 API를 사용하여 플레이어 구성 개체를 가져올 때 반환된 데이터에 플레이어를 HTML 페이지에 포함하기 위한 iframe 태그와 플레이어를 직접 포함하려는 경우 전체 HTML에 대한 URL을 포함하는 것을 보았습니다. 페이지.

가능한 한 iframe 삽입을 사용하는 것이 좋습니다. 이렇게 하면 플레이어의 CSS와 JavaScript, 페이지에서 사용되는 다른 스타일시트 및 스크립트 간의 충돌을 방지할 수 있습니다. 보다이 문서차이점에 대한 자세한 내용과 다양한 상황에서 사용할 임베딩에 대한 보다 구체적인 권장 사항은

어떤 임베딩을 선택하든 플레이어에 콘텐츠를 추가하려면 임베드 코드에 Video Cloud 비디오 ID 또는 재생 목록 ID를 추가해야합니다. iframe 임베드 코드는 다음과 같습니다.

    <iframe
    src='//players.brightcove.net/57838016001/de055fa4-4f09-45af-8531-419c6794ad04_default/index.html'
    allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>

플레이어의 URL에 매개 변수를videoId={}video_id추가해야 전체 임베드 코드가 다음과 같이 표시됩니다.

    <iframe
    src='//players.brightcove.net/57838016001/de055fa4-4f09-45af-8531-419c6794ad04_default/index.html?videoId=4483119716001'
    allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>

재생 목록 플레이어인 경우 매개 변수를playlistId={playlist_id}대신 사용합니다. 인 페이지 소스 코드의 수정도 비슷합니다.

플레이어 구성에서 플레이어 크기가 고정되지 않는 한, style어트리뷰트에 너비와 높이를 추가하여 플레이어의 크기를 조정해야 합니다.

    <iframe
    src='//players.brightcove.net/57838016001/de055fa4-4f09-45af-8531-419c6794ad04_default/index.html?videoId=4483119716001'
    allowfullscreen webkitallowfullscreen mozallowfullscreen
    style=width:640px;height:360px;></iframe>
