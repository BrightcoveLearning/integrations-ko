보고 데이터베이스 구성

비디오 클라우드Sitecore미디어 프레임워크 보고서가 작동하려면 경험 데이터베이스 (xDB) 보고 데이터베이스에 새 팩트 테이블을 추가해야 합니다. 이 새 테이블을 만드는 스크립트는 SDN에서 사용할 수 있습니다. 이 스크립트는 xDB 보고 데이터베이스에서 실행해야 합니다. 이 단계는 데이터베이스 관리자가 완료해야합니다.

참고: 여러 개의비디오 클라우드사이트코어미디어 프레임워크 커넥터를 설치할 수 있습니다. 이 스크립트는 설치 한 커넥터 수에 관계없이보고 데이터베이스 당 한 번만 실행하면됩니다.

다중 서버 구성

비디오 클라우드Sitecore미디어 프레임워크는 다중 서버 콘텐츠 제작 환경에 설치할 수 있습니다. 이러한 방식으로비디오 클라우드Sitecore환경이 구성된 경우 다음을 구성해야 합니다.

콘텐츠 관리 서버

스케줄러를 사용하여 미디어를 동기화하는 경우 하나의 CM (Content Management) 서버에만 스케줄러를 구성하면됩니다. 다른 CM 서버의 구성 파일에서 예약 된 작업을 제거하십시오. 비디오 클라우드Sitecore이벤트 대기열은 다른 CM 서버의 동기화 상태를 유지합니다.

참고: 여러 CM 서버에서 스케줄러를 구성해도 문제가 발생하지는 않지만 그렇게해도 이점은 없습니다.

콘텐츠 전달 서버

각 CD (콘텐츠 전송) 서버에서 지정된 구성 파일에서 다음 인덱스를 제거합니다. 이 색인은 CM 환경에서만 필요합니다.

파일: Video Cloud Sitecore .MediaFramework.Services.Brightcove.config
인덱스 이름: MediaFramework_Brightcove_Index
 

CD 서버를 구성 할 때 CM 서버에서 구성 파일을 복사하는 것이 편리 할 수 있습니다. 이 작업을 수행하고 미디어를 동기화하는 스케줄러를 추가 한 경우 CD 서버의 구성 파일에서 예약 된 작업을 제거해야합니다. 미디어 동기화는 CD 서버에서 실행되지 않아야합니다. 하나의 CM 서버에서 실행되어야합니다. 게시는 CD 서버에서 미디어 항목을 사용할 수 있도록하는 데 사용됩니다.

비디오 클라우드계정 연결

이 항목에서는 다음에서 계정을 만드는 방법을 다룹니다. 비디오 클라우드사이트코어 Brightcove에 해당하는비디오 클라우드계정.

 

Brightcove에서 계정 자격 증명 받기

미디어 프레임워크가 브라이트코브비디오 클라우드에연결하려면 브라이트코브의 계정 자격 증명을비디오 클라우드사이트코어에 추가해야 합니다. . 이 섹션에서는 Brightcove에서 계정 자격 증명을 찾는 방법을 설명합니다. 다음 섹션에서는 이러한 자격 증명을 Video Cloud Sitecore에 추가하는 방법에 대해 설명합니다 .

  1. 브라이트코브비디오 클라우드에로그인합니다. 게시자 ID 값을 기록해 둡니다. 다음 섹션에서비디오 클라우드사이트코어에 입력해야 합니다.
  2. 관리자로 이동합니다.
  3. API 인증으로 이동합니다.
  4. 새 응용 프로그램을 등록하십시오.
  5. 애플리케이션 이름을 지정하고 계정 액세스를 설정하고 API를 활성화합니다.
  6. 클라이언트 ID와 클라이언트 시크릿을 기록해 둡니다. 다음 섹션에서비디오 클라우드사이트코어에 입력해야 합니다.

비디오 클라우드사이트코어에 브라이트코브 계정 추가

이 섹션에서는비디오 클라우드사이트코어에 브라이트코브 계정을 추가하는 방법에 대해 설명합니다 .

  1. 비디오 클라우드사이트코어데스크톱에 로그인합니다.
  2. 미디어 라이브러리를 엽니 다.
  3. 미디어 라이브러리> 미디어 프레임 워크> 계정으로 이동합니다.
  4. 새 Brightcove 계정 항목을 추가하십시오.
  5. 이전 섹션에서 기록한 게시자 ID, 클라이언트 ID 및 클라이언트 비밀 값을 입력합니다.
  6. 항목을 저장하십시오.

비디오 클라우드플레이어 정의

단일 비디오 또는 비디오 재생 목록을 표시하려면 플레이어가 필요합니다. 플레이어는 비디오를 보려는 방문자가 사용할 수있는 기능의 종류를 정의합니다. 비디오 클라우드Sitecore미디어 프레임워크를 사용하면 콘텐츠 작성자가 새로운 플레이어를 추가하고 기존 플레이어를 편집하고 기존 플레이어를 삭제할 수 있습니다.

 

플레이어 추가

이 섹션에서는 플레이어에 해당하는 Video Cloud Sitecore항목을 수동으로 만드는 방법에 대해 설명합니다.

  1. 브라이트코브비디오 클라우드에서플레이어 섹션으로 이동합니다.
  2. 비디오 클라우드사이트코어에서 만들려는 플레이어를 선택합니다 . 플레이어 ID 값을 기록해 둡니다. 다음 섹션에서비디오 클라우드사이트코어에 입력해야 합니다.
  3. 비디오 클라우드사이트코어의미디어 라이브러리에서 브라이트코브 계정 항목을 선택합니다.
  4. 계정 항목을 선택하십시오.
  5. 플레이어 항목을 선택하십시오.
  6. 새 Brightcove 비디오 플레이어 항목을 추가하십시오. 플레이어 ID를 입력하라는 메시지가 표시됩니다. 브라이트코브비디오 클라우드의 값을 입력합니다 .

기본 플레이어 설정

Video Cloud Sitecore에서 비디오 및 재생 목록을 미리 보기 위해서는 기본 플레이어를 지정해야 합니다. 이 섹션에서는 기본 플레이어를 설정하는 방법을 설명합니다.

  1. 비디오 클라우드사이트코어의미디어 라이브러리에서 브라이트코브 계정 항목을 선택합니다.
  2. 계정 항목을 선택하십시오.
  3. 설정 항목을 선택합니다.
  4. 기본 비디오 플레이어 필드 및 기본 재생 목록 플레이어 필드의 값을 설정합니다.
  5. 항목 저장

플레이어 편집

플레이어 항목은 다른비디오 클라우드사이트코어항목과 마찬가지로 편집됩니다.

플레이어 삭제

플레이어 항목은 다른비디오 클라우드사이트코어항목과 마찬가지로 삭제됩니다.

참고: 비디오 클라우드사이트코어에서플레이어를 삭제해도 브라이트코브비디오 클라우드에서 플레이어가 삭제되지는 않습니다 .

비디오 가져 오기, 추가, 미리보기 및 삭제

비디오 클라우드Sitecore미디어 프레임워크를 사용하면 콘텐츠 작성자가 자신의 로컬 컴퓨터에서 비디오를 업로드하고 기존 비디오를 편집 및 삭제할 수 있습니다. 비디오 콘텐츠를 구성하기 위해 새 태그를 정의 할 수 있습니다.

 

비디오 가져 오기

Video Cloud Sitecore콘텐츠 작성자 또는 관리자는 미디어 가져오기 프로세스를 수동으로 트리거할 수 있습니다.

비디오를 가져올 때 Video Cloud Sitecore에 동영상 내 업로드 날짜가 아니라 가져온 현재 날짜가반영됩니다. 비디오 클라우드 . 기존 Video Cloud Sitecore구현을 이동하는 경우 이전 콘텐츠를 마이그레이션하면 커넥터가 이러한 비디오를 인식하게 됩니다.

Brightcove Video Connect 플러그인에는 정기적으로 미디어를 자동으로 동기화하는 에이전트가 포함되어 있습니다. 에이전트는 다음 위치에 정의됩니다.

구성 파일: Video Cloud Sitecore .MediaFramework.Services.Brightcove.config
에이전트 이름: MediaFramework_Import_Brightcove

  2. 미디어 프레임 워크 리본을 클릭합니다.
  3. 모두 가져 오기 버튼을 클릭합니다.
  4. 가져 오기 프로세스가 실행되면 상태 상자가 나타납니다. 상태 상자가 사라지면 가져 오기 프로세스가 완료된 것입니다.
  5. 미디어를 가져 왔는지 확인하려면 검색 탭을 클릭합니다.

비디오 추가

비디오 클라우드Sitecore미디어 프레임워크를 사용하면 콘텐츠 작성자가 새 비디오를 추가할 수 있습니다.

  1. 비디오 클라우드사이트코어미디어 라이브러리를 엽니다.
  2. 재생 목록을 만들려는 동영상이있는 계정으로 이동합니다.
  3. 미디어 콘텐츠 항목으로 이동합니다.
  4. Brightcove 비디오를 클릭합니다.
  5. 파일 추가를 클릭합니다.
  6. 로컬 컴퓨터에서 업로드 할 파일을 선택합니다.
  7. 시작 을 클릭합니다.
  8. 동영상이 업로드되면 업 로더 창을 닫습니다.

재생 목록을 가져오면 Video Cloud Sitecore에 재생 목록을 가져온 현재 날짜가 아닌 내부 생성 날짜가반영됩니다. 비디오 클라우드 . 기존 Video Cloud Sitecore구현을 이동하는 경우 이전 콘텐츠를 마이그레이션하면 커넥터가 이러한 비디오를 인식하게 됩니다.

리치 텍스트 편집기

서식있는 텍스트 필드를 사용하면 콘텐츠 작성자가 서식이 지정된 콘텐츠를 페이지에 삽입 할 수 있습니다. 콘텐츠 작성자는 서식있는 텍스트 편집기를 사용하여이 형식이 지정된 콘텐츠를 만들고 편집합니다. 이 섹션에서는 Video Cloud Sitecore미디어 프레임워크가 서식 있는 텍스트 편집기에 추가하는 기능에 대해 설명합니다.

미디어 삽입

리치 텍스트 필드에 미디어를 직접 삽입 할 수 있습니다.

  1. 서식있는 텍스트 편집기에서 미디어 삽입 버튼을 클릭합니다.
  2. 검색 화면을 사용하여 삽입 할 미디어를 찾습니다. 삽입 할 미디어를 나타내는 행을 클릭합니다. 다음> 버튼을 클릭합니다.
  3. 선택한 미디어 유형에 따라 추가 옵션이 표시됩니다. 예를 들어 비디오를 선택하는 경우 플레이어의 크기와 함께 비디오를 표시하는 데 사용할 플레이어를 지정해야합니다. 마침을 클릭합니다.
  4. 미디어는 서식있는 텍스트 필드에 포함됩니다.

미디어 링크 포함

미디어 링크를 서식있는 텍스트 필드에 삽입 할 수 있습니다.

  1. 서식있는 텍스트 편집기에서 미디어 링크 포함 버튼을 클릭합니다.
  2. 검색 화면을 사용하여 삽입 할 미디어를 찾습니다. 삽입 할 미디어를 나타내는 행을 클릭합니다. 다음> 버튼을 클릭합니다.
  3. 링크 제목에 대한 값을 입력하십시오. 이것은 미디어 링크에 표시되는 텍스트입니다.
    또한 선택한 미디어 유형에 따라 추가 옵션이 표시됩니다. 예를 들어 비디오를 선택하는 경우 플레이어의 크기와 함께 비디오를 표시하는 데 사용할 플레이어를 지정해야합니다. 마침을 클릭합니다.
  4. 서식있는 텍스트 필드에 링크가 포함됩니다.

렌더링을 통해 콘텐츠를 페이지의 특정 위치에 배치 할 수 있습니다. 비디오 클라우드Sitecore미디어 프레임워크에는 콘텐츠 작성자가 페이지에 미디어를 삽입할 수 있는 렌더링이 포함되어 있습니다.

미디어 렌더링 포함

  1. 페이지 편집기를 사용하여 자리 표시 자에 렌더링을 추가합니다.
  2. Embed Media 렌더링을 선택합니다.
  3. 렌더링이 자리 표시 자에 추가되었습니다. 이제 미디어 요소 포함 버튼을 클릭합니다.
  4. 검색 화면을 사용하여 삽입 할 미디어를 찾습니다. 삽입 할 미디어를 나타내는 행을 클릭합니다. 다음> 버튼을 클릭합니다.
  5. 선택한 미디어 유형에 따라 추가 옵션이 표시됩니다. 예를 들어 비디오를 선택하는 경우 플레이어의 크기와 함께 비디오를 표시하는 데 사용할 플레이어를 지정해야합니다. 마침을 클릭합니다.
  6. 렌더링 할 미디어가 선택되었습니다.

비디오 클라우드사이트코어와비디오 클라우드 간의 동기화

이 섹션에서는 다음과 같은 시나리오에서비디오 클라우드Sitecore간에 비디오 및 재생 목록을 동기화하는 방법을 다룹니다.

  • Brightcove에서 비디오 및 재생 목록이 삭제 된 경우
  • 동영상과 재생목록이 추가되었을 때비디오 클라우드사이트코어
  • 비디오 및 재생 목록이 다음에서 삭제될 때비디오 클라우드사이트코어

Brightcove에서 다음으로 가져오기비디오 클라우드사이트코어이전에 다룬여기 .

비디오 내보내기

비디오 클라우드사이트코어내에서 브라이트코브 미디어는비디오 클라우드사이트코어를 사용하여 액세스합니다. 미디어 가져오기 프로세스 중에 만들어진 항목입니다. Video Cloud Sitecore항목이므로 사용자가 해당 항목을 변경할 수 있습니다. 미디어 내보내기 프로세스는비디오 클라우드사이트코어에서 브라이트코브로 변경 사항을 내보내는작업을 담당합니다.

자동 미디어 내보내기

브라이트코브 미디어에 해당하는비디오 클라우드사이트코어항목이비디오 클라우드에서 생성, 저장 또는 삭제된 경우사이트코어클라이언트에서는 이러한 변경 사항이 브라이트코브로 푸시됩니다.

실시간 내보내기

Video Cloud Sitecore에서 새 미디어 항목이 생성되면 기본적으로 실시간 내보내기가 수행됩니다. 즉, 내보내기 작업이 즉시 수행됩니다.

자동 내보내기 방지

비디오 클라우드사이트코어에서 변경한 내용을 브라이트코브로 내보내지 않을 수 있습니다. 비디오 클라우드사이트코어에서브라이트코브로의 연결을 읽기 전용으로 설정해야 할 수 있습니다. 여기서 브라이트코브의 변경 사항을비디오 클라우드로 가져옵니다.사이트코어이지만비디오 클라우드사이트코어변경 사항이 브라이트코브로 내보내지지 않습니다.

비디오 클라우드사이트코어에서 변경한 내용이 브라이트코브로 내보내지 않도록 하려면 다음 설정 값을 지웁니다.

구성 파일: Video Cloud Sitecore .MediaFramework.config
설정 이름: Video Cloud Sitecore .MediaFramework.ExportDatabases
 

참고: 이 설정은 브라이트코브비디오 연결 플러그인뿐만 아니라 비디오 클라우드사이트코어미디어 프레임워크에 적용됩니다. 이 설정을 변경하면 모든 커넥터와 모든 계정에 적용됩니다.

데이터 정리

정리 프로세스는비디오 클라우드사이트코어와 브라이트코브 데이터를 동기화하는 데 사용됩니다.

비디오 클라우드사이트코어정리

비디오 클라우드사이트코어정리는 브라이트코브 및 해당비디오 클라우드에서 미디어가 삭제될 때 실행되는 프로세스입니다.사이트코어항목을 삭제해야 합니다. 브라이트코브는비디오 클라우드사이트코어에대해 아무 것도 알지 못하기 때문에 브라이트코브는비디오 클라우드사이트코어에 알릴 수 없습니다.해당 미디어가 삭제되었습니다. 따라서비디오 클라우드사이트코어는 사전예방적이어야 합니다.

자동 정리

Brightcove Video Connect 플러그인에는 정기적으로 정리 프로세스를 자동으로 실행하는 에이전트가 포함되어 있습니다. 에이전트는 다음 위치에 정의됩니다.

구성 파일: Video Cloud Sitecore .MediaFramework.Services.Brightcove.config
에이전트 이름: MediaFramework_Cleanup_Brightcove

Brightcove 정리

브라이트코브 정리는비디오 클라우드사이트코어에서 미디어 항목이 삭제되고 브라이트코브의 해당 미디어를 삭제하거나 비활성화해야 할 때 실행되는 프로세스입니다. 브라이트코브는 미디어 항목이 삭제되었음을 알고 있기 때문에비디오 클라우드사이트코어는 브라이트코브의 해당 미디어를 삭제하거나 비활성화해야 함을 브라이트코브에 알릴 수 있습니다.

Video Cloud Sitecore클라이언트에서 미디어 항목이 삭제되면 내보내기 대기열에 항목이 만들어집니다. 프로세스는 미디어 항목을 저장할 때와 동일합니다.

재생 이벤트 분석

비디오 클라우드Sitecore미디어 프레임워크를 사용하면 비디오 클라우드에서 사용하기 위해 비디오 재생 이벤트를 기록할 수 있습니다.사이트코어디지털 마케팅 시스템 (DMS). 이 섹션에서는 사용 가능한 옵션에 대해 설명합니다.

재생 이벤트

재생 이벤트는 비디오 플레이어 작업의 결과로 DMS 페이지 이벤트를 기록하는 데 사용됩니다. 예를 들어 비디오 재생이 시작되면 DMS 페이지 이벤트를 작성할 수 있습니다.

기본 재생 이벤트

비디오 클라우드사이트코어미디어 프레임워크는 다음과 같은 재생 이벤트와 함께 제공됩니다.

  • 재생 시작됨
  • 재생 완료
  • 재생 오류

계정에 대한 재생 이벤트 구성

기본적으로 재생 시작 및 재생 완료 이벤트가 기록됩니다. 이는 Brightcove 계정 항목 아래의 설정 항목에서 활성화됩니다.

설정 항목에는 재생 이벤트 규칙이라는 필드가 있습니다. 비디오 및 재생 목록에 지정된 이벤트를 기록하도록 미디어 프레임 워크에 지시하는 규칙이 정의됩니다.

계정에 대한 재생 이벤트 비활성화

특정 계정에 대해 모든 재생 이벤트를 비활성화 할 수 있습니다.

  1. 콘텐츠 편집기에서 미디어 라이브러리> 미디어 프레임 워크> 계정 아래의 계정으로 이동합니다.
  2. 설정 항목을 선택합니다.
  3. 재생 이벤트 규칙 필드는 재생 이벤트를 등록하기 위해 충족해야하는 조건을 설명합니다. 기본적으로 조건은 "항상"을 의미하는 "true"입니다. 조건을 변경하거나 기존 규칙을 제거하려면 규칙 편집 단추를 클릭하십시오.

미디어에 대한 재생 이벤트 지정

각 재생 목록 및 비디오 항목에는 이벤트라는 필드가 있습니다. 이 필드는 지정된 항목에 대해 처리 할 재생 이벤트를 결정합니다.

다음 스크린 샷은 비디오에 대해 재생 이벤트가 지정되었음을 나타냅니다.

참고: 표준 값을 사용하여 모든 재생 목록 및 비디오에 대해 특정 이벤트가 기록되도록 할 수 있지만이 필드에서는 개별 재생 목록 및 비디오에서 자체 이벤트를 선택할 수도 있습니다.

참고: 이러한 이벤트는 Brightcove 계정 항목 아래의 설정 항목에 "미디어 항목 이벤트 추가"작업으로 구성된 재생 이벤트 규칙 필드가있는 경우에만 기록됩니다. 섹션 3.1.2를 참조하십시오.

새 재생 이벤트 생성

비디오 재생에 대한 자세한 정보를 캡처하기 위해 새 재생 이벤트를 추가 할 수 있습니다.

  1. 콘텐츠 편집기에서 시스템> 모듈> 미디어 프레임 워크> 재생 이벤트로 이동합니다.
  2. 템플릿 재생 이벤트를 사용하여 새 항목 추가
  3. 페이지 이벤트 필드에서 재생 변경을 선택하십시오.
  4. 매개 변수 필드에 다음 형식 중 하나로 값을 입력 할 수 있습니다.
    1. 숫자-이 시간 (초) 동안 비디오가 재생 된 후 페이지 이벤트가 기록됩니다. 유효한 값의 예는 15입니다.
    2. 백분율-동영상이이 양으로 재생 된 후 페이지 이벤트가 기록됩니다. 유효한 값의 예는 50 %입니다.
  5. 항목 저장
  6. 항목 게시
  7. 이제 재생 목록과 동영상에 새 페이지 이벤트를 추가 할 수 있습니다.

참고: 새 재생 이벤트를 재생 목록과 비디오에 할당해야합니다. 그렇지 않으면 이벤트가 기록되지 않습니다.

이 보고서는 동영상 완료 이벤트를 트리거 한 동영상 목록을 보여줍니다.
