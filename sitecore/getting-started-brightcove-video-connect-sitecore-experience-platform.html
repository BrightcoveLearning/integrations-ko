    보고 데이터베이스 구성

    위해Video CloudSitecore미디어 프레임 워크가 작동하려면 xDB (Experience Database)보고 데이터베이스에 새 팩트 테이블을 추가해야합니다. 이 새 테이블을 만드는 스크립트는 SDN에서 사용할 수 있습니다. xDB 보고 데이터베이스에서 이 스크립트를 실행해야 합니다. 이 단계는 데이터베이스 관리자가 완료해야 합니다.

    참고: 다중 설치 가능Video CloudSitecore미디어 프레임 워크 커넥터. 이 스크립트는 설치 한 커넥터 수에 관계 없이 보고 데이터베이스 당 한 번만 실행 해야 합니다.

    다중 서버 구성

    Video CloudSitecore미디어 프레임 워크는 다중 서버 콘텐츠 제작 환경에 설치할 수 있습니다. 귀하의Video CloudSitecore환경이 이러한 방식으로 구성되면 다음을 구성해야합니다.

    컨텐츠 관리 서버

    스케줄러를 사용하여 미디어를 동기화하는 경우 하나의 CM (콘텐츠 관리) 서버만 스케줄러를 구성하면 됩니다. 다른 CM 서버의 구성 파일에서 예약된 작업을 제거합니다. 그만큼Video CloudSitecore이벤트 큐는 다른 CM 서버의 동기화 유지를 처리합니다.

    참고: 여러 CM 서버에서 스케줄러를 구성해도 문제가 발생하지 않지만 이렇게 하면 이점이 없습니다.

    콘텐츠 전송 서버

    각 콘텐츠 배달 (CD) 서버에서 지정된 구성 파일에서 다음 인덱스를 제거합니다. 이 인덱스는 CM 환경에서만 필요합니다.

    파일: Video Cloud Sitecore .MediaFramework.Services.Brightcove.config
    인덱스 이름: MediaFramework_Brightcove_Index
     

    CD 서버를 구성할 때 CM 서버에서 구성 파일을 복사하는 것이 편리할 수 있습니다. 이렇게 하고 미디어를 동기화하는 스케줄러를 추가한 경우 CD 서버의 구성 파일에서 예약된 작업을 제거해야 합니다. CD 서버에서 미디어 동기화를 실행해서는 안 됩니다. 하나의 CM 서버에서 실행되어야 합니다. 게시는 CD 서버에서 미디어 항목을 사용할 수 있도록 하는 데 사용됩니다.

    기본 플레이어 설정

    특정 계정에 대해 모든 재생 이벤트를 비활성화할 수 있습니다.

    1. 컨텐츠 편집기에서 미디어 라이브러리> 미디어 프레임 워크> 계정 아래에서 계정으로 이동합니다.
    2. 설정 항목을 선택합니다.
    3. 재생 이벤트 규칙 필드는 재생 이벤트를 등록하기 위해 충족되어야 하는 조건을 설명합니다. 기본적으로 조건은 “true”이며 “항상”을 의미합니다. 규칙 편집 단추를 클릭하여 조건을 변경하거나 기존 규칙을 제거합니다.

    미디어의 재생 이벤트 지정

    각 재생 목록 및 비디오 항목에는 Events라는 필드가 있습니다. 이 필드는 지정된 항목에 대해 처리될 재생 이벤트를 결정합니다.

    다음 스크린샷은 비디오에 대해 재생 이벤트가 지정되었음을 나타냅니다.

    참고: Standard Values는 모든 재생목록과 동영상에 대해 특정 이벤트가 녹화되도록 하는 데 사용할 수 있지만, 이 필드에서는 개별 재생목록과 동영상에 고유한 이벤트를 선택할 수도 있습니다.

    참고: 이러한 이벤트는 브라이트코브 계정 항목의 설정 항목에 “미디어 항목 이벤트 추가” 동작으로 구성된 재생 이벤트 규칙 필드가 있는 경우에만 기록됩니다. 3.1.2 절을 참고하십시오.

    새 재생 이벤트 만들기

    비디오 재생에 대한 자세한 정보를 캡처하기 위해 새 재생 이벤트를 추가할 수 있습니다.

    1. 컨텐츠 편집기에서 시스템> 모듈> 미디어 프레임 워크> 재생 이벤트로 이동합니다.
    2. 재생 이벤트 템플릿을 사용하여 새 항목 추가
    3. 페이지 이벤트 필드에서 재생이 변경됨을 선택합니다.
    4. 매개변수 필드에 다음 형식 중 하나로 값을 입력할 수 있습니다.
      1. 번호 - 비디오가이 시간 (초) 을 재생 한 후 페이지 이벤트가 기록됩니다. 유효한 값의 예는 15입니다.
      2. 백분율 - 비디오가이 금액을 재생 한 후 페이지 이벤트가 기록됩니다. 유효한 값의 예는 50% 입니다.
    5. 항목 저장
    6. 항목 게시
    7. 이제 재생 목록과 비디오에 새 페이지 이벤트를 추가 할 수 있습니다

    참고: 재생 목록과 비디오에 새 재생 이벤트를 할당해야합니다. 그렇지 않으면 이벤트가 기록되지 않습니다.
