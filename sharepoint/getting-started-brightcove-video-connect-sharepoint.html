SharePoint 용 Brightcove Video Connect 설치

설치를 시작하기 전에 필수 구성 요소 및 권한에 대한 다음 정보를 검토하십시오.

  • 사용자가 SharePoint 용 앱을 추가하려면 Farm Administrators 그룹의 구성원이 SharePoint 용 앱을 지원하도록 환경을 구성해야합니다. 자세한 내용은 SharePoint 2013용 앱 환경 구성또는셰어포인트 2016 또는 셰어포인트 2019 .
  • SharePoint용 앱을 추가하려면 사용자에게 웹 사이트관리및 하위 사이트만들기권한이 있어야 합니다. 기본적으로 이러한 권한은 모든 권한 수준이 있거나 사이트 소유자 그룹에 속한 사용자 만 사용할 수 있습니다.
  • 사용자가 SharePoint용 앱을 추가하면 앱이 작동하는 데 필요한 권한 (예: 검색 액세스 또는 목록 만들기) 을 요청합니다. 해당 권한이없는 사용자에게는 충분한 권한이 없으며 앱을 추가 할 수 없다는 메시지가 표시됩니다. 사용자는 사이트 또는 팜 관리자에게 연락하여 관리자가 앱을 추가 할 수 있는지 확인할 수 있습니다.
  • 시스템 계정으로 사이트에 로그인 한 사용자는 앱을 설치할 수 없습니다. 시스템 계정은 성능 문제가 발생할 수 있으므로 앱 라이선스를 가져올 수 없습니다.

앱 카탈로그에 Brightcove Video Connect 앱 추가

앱 카탈로그 설정

Brightcove 커넥터를 설치하려면 첫 번째 단계는 SharePoint에서 앱 카탈로그를 설정하는 것입니다. 이 단계를 완료하려면 아래 참조를 사용하십시오.

  • SharePoint 2013에서 앱 카탈로그 관리
  • SharePoint 2016에서 앱 카탈로그 관리
  • SharePoint 2019에서 앱 카탈로그 관리

카탈로그에 Brightcove Video Connect 앱 추가

앱 카탈로그가 설정되면 아래 나열된 단계를 따르십시오.

  1. 이 절차를 수행하는 사용자 계정이 앱 카탈로그에 대한 사이트 소유자 또는 디자이너 그룹의 구성원인지 확인합니다.
  2. 앱 카탈로그 사이트에서 SharePoint용앱목록을 클릭합니다.
  3. SharePoint용앱페이지에서새 항목을 클릭합니다 .
  4. 파일선택상자에서찾아보기를클릭한 다음 브라이트코브커넥터앱 파일.
  5. 앱을 선택한 다음열기를 클릭합니다 .
  6. 확인을클릭하여 앱을 업로드합니다.
  7. 항목 세부 정보 상자에 앱에 대한 이름, 제목, 간단한 설명, 아이콘 URL 및 기타 설정을 제공합니다. 사용자가 사이트에서 앱을 볼 수 있도록활성화됨확인란이 선택되어 있는지 확인합니다. 추천 확인란을 선택하여 앱 카탈로그의추천콘텐츠 보기에 앱을 나열할 수 있습니다.
  8. 저장을 클릭합니다 .

사이트에 Brightcove Video Connect 앱 설치

아래 나열된 단계에 따라 사이트에 Brightcove 커넥터를 설치하십시오.

  1. 이 절차를 수행하는 사용자 계정이 사이트 소유자 그룹의 구성원인지 확인합니다.
  2. 다음 두 가지 방법 중 하나로 앱을 사이트에 추가 할 수 있습니다.
    1. 모든 페이지의설정메뉴에서 앱추가옵션을 선택합니다.
    2. 사이트콘텐츠에서 앱추가옵션을 사용합니다.
  3. 그런 다음 “조직에서”를 클릭한 다음 BrightCoveConnector앱을 클릭합니다.
  4. 권한 부여 대화 상자에서 "신뢰"를 클릭하십시오.
  5. 이 시점에서 SharePoint용 BrightCoveConnector앱이 추가되어 사이트 콘텐츠 목록의 앱 섹션에 나타납니다. 이제 사이트에서 사용할 수 있습니다.

Brightcove Video Connect 앱에 대한 기본 목록 프로비저닝

Brightcove 커넥터를 사용하기 전에 아래 단계에 따라 일부 필수 목록을 프로비저닝해야합니다.

  • 앱이 처음 인스턴스화될 때 또는 목록이 로컬 사이트에 없는 경우 사이트 내에서 목록을 프로비저닝할 수 있도록 사용자에게 최대 3개의 옵션이 제공됩니다.

  • 클릭하면 각 버튼에 효과에 대한 메시지가 표시됩니다.
  • "설정 목록 만들기"를 클릭하면 Brightcove 설정 목록을 만드는 것 외에도 이 목록을 편집하라는 메시지가 표시되어 Brightcove API 지원에 필요한 프록시 URL이 구성되었는지 확인합니다.
  • 이 링크를 클릭하면 프록시 서버 액세스 정보를 즉시 편집 할 수있는 Brightcove 설정 목록이 열립니다. 제목은 "프록시"로 설정되어야합니다. ProxyURL은 Brightcove API에 액세스하는 데 사용할 프록시 서버의 위치에 따라 다릅니다. 프록시 서버사용에대한 자세한 내용은 브라이트코브 API 통신에 프록시 서버 사용을 참조하십시오.
  • 목록 프로비저닝을 완료하면 사이트에 다음 목록이 생성됩니다.
    • 계정-계정 개체를 나타내는 사용자 지정 목록입니다.
    • 비디오-비디오 개체를 나타내는 사용자 지정 목록입니다.
    • 설정-프록시 서버 세부 정보를 저장할 목록입니다.

SharePoint Online 용 플레이어 솔루션 패키지 설치

Brightcove Video Connect 패키지에 포함 된 Brightcove 플레이어 솔루션 패키지를 사용하면 사용자가 SharePoint 사이트 페이지에 Brightcove 비디오 플레이어를 포함 할 수 있습니다. Brightcove Player 솔루션 패키지는 SharePoint Online 및 SharePoint Server와 모두 호환되므로 즉시 사용할 수있는 SharePoint 컨트롤을 사용합니다. 비디오 포함을위한 플레이어 웹 파트 샌드 박스 솔루션은 Brightcove Video Connect 패키지에서도 사용할 수 있지만 Microsoft에서 호스팅하는 SharePoint Online 및 SharePoint Server에는 샌드 박스 솔루션을 금지하는 보안 제한이 있으므로 온-프레미스에서 SharePoint Server에서만 사용할 수 있습니다. 이 솔루션은 다음 섹션에서 설명합니다.

SharePoint Online에 Brightcove 플레이어를 포함하려면 여기에 설명 된 솔루션 패키지를 통해 사용자가 비디오 또는 재생 목록 플레이어를 선택한 다음 플레이어 포함 코드는 사용자가 SharePoint 포함 코드 대화 상자에 복사 / 붙여 넣기 할 수 있도록 생성됩니다.

참고-이 섹션에는 BrightcovePlayer.wsp 솔루션 파일이 필요합니다.

  1. Brightcove 앱이 설치된 사이트 모음의 루트 사이트에서 사이트 설정 -> 사이트 모음 관리 -> HTML 필드 보안.
  2. 허용된 iframe 도메인으로players.brightcove.net 추가
  3. Brightcove 앱이 설치된 사이트 모음의 루트 사이트에서 사이트 설정 -> 웹 디자이너 갤러리 -> 솔루션.
  4. "BrightcovePlayer.wsp" 파일을 솔루션 갤러리에 업로드하고 파일을 활성화합니다.
  5. 사이트 설정으로 이동 -> 사이트 작업 -> 사이트 기능을 관리합니다.
  6. "Brightcove Player" 기능을 활성화합니다.
  7. 설치 중에 BrightcoveConnectorResources 문서 라이브러리가 생성되었는지 확인합니다.
  8. 그렇지 않은 경우 BrightcoveConnectorResources라는 새 문서 라이브러리를 만듭니다.
  9. BrightcovePlayer.css 및 BrightcovePlayer.js 를 해당 문서 라이브러리에 업로드합니다.

Brightcove API 통신을위한 프록시 서버 설치

커넥터는 클라이언트 측 JavaScript 애플리케이션이므로 인증 된 통신을 Brightcove API에 중개하려면 프록시 서버가 필요합니다. 자세한 내용은브라이트코브 API 문서에서 확인할 수 있습니다.

.net 프록시 앱은 Brightcove Video Connect 패키지에 제공됩니다. 프록시 서버는 Microsoft Azure와 같이 웹에 액세스 할 수있는 위치에 배포해야합니다. 보안상의 이유로 프록시는 프록시와 함께 설치된 web.config 파일에 포함 된 웹 사이트의 연결 만 허용하도록 설계되었습니다. web.config 파일의 브라이트코브:오리진 키에는 BrightCoveConnector앱의 SharePoint 웹 사이트 URL과브라이트코브플레이어가 있는 사이트의 웹 사이트 URL이 포함되어야 합니다.구성 요소가 실행됩니다. 아래 예를 참조하십시오.

<add key="brightcove:origins" value="https://brightcovecms-a3da847a1c9e33.sharepoint.com, https://brightcovecms.sharepoint.com" />

여러 사이트를 쉼표로 구분 된 문자열로 나열 할 수 있습니다.

Brightcove Video Connect for SharePoint 시작

응용 프로그램을 시작하려면 사이트 콘텐츠 목록에서 BrightCoveConnector앱을 클릭하거나 SharePoint 관리자가 설정한 경우 메뉴 링크를 클릭할 수 있습니다.

두 경우 모두 사용자는 Connector 앱의 홈 페이지로 이동합니다. 이 페이지는 추가 회사 별 메시지, 추가 링크 등을 포함하도록 클라이언트가 추가로 구성 할 수 있습니다. 설치 섹션에 설명 된대로 사용자의 그룹 구성원에 따라 계정 관리 및 비디오 추가 탭이 표시되거나 표시되지 않을 수 있습니다.

커넥터의 왼쪽 열에 있는 기본 탐색 기능은 항상 존재하며 사용자가 액세스할 수 있는 모든 섹션을 탐색할 수 있습니다 ( 사용 권한에 관한 위의 섹션 참조 ). 다음은 커넥터의 각 구성 요소 목적에 대한 간략한 설명입니다.

  • 집 -커넥터에 처음 들어갈 때의 랜딩 페이지입니다.
  • 계정 관리 -Brightcove Video Cloud 계정 정보를 SharePoint에 입력하기위한 인터페이스를 제공합니다.
  • 비디오 추가 -비디오 클라우드에 새 비디오를 업로드합니다. SharePoint에서도 비디오 개체가 생성됩니다 (실제 비디오 파일 제외).
  • 비디오 관리 -기존 비디오를 검색 및 편집하고 Video Cloud에서 SharePoint로 비디오를 가져올 수 있습니다.
  • 재생 목록 관리 -새 재생 목록을 만들고 기존 재생 목록을 탐색하고 관리합니다.

브라이트코브에서 비디오추가및 비디오편집구성 요소도 있습니다. 이 구성 요소는 이러한 작업을 수행할 때 기본 탐색에 표시됩니다.

Video Cloud 계정에 연결

Connector의 계정 관리 섹션은 사이트에서 액세스 할 연결된 각 Brightcove Video Cloud 계정에 대한 정보를 입력하는 데 사용됩니다. Brightcove 계정은 SharePoint에서 만들 수 없으며 먼저 Video Cloud에서 만들어야합니다. Brightcove 계정 정보는 비디오 클라우드와 통신하기 위해 Brightcove API에 액세스 할 때 사용되므로 SharePoint에 입력해야합니다. 또한 계정에 특정한 액세스 권한을 할당하는 방법을 제공합니다. 다중 계정 엔터프라이즈 클라이언트의 경우 각 계정에 액세스 할 수있는 사용자와 비디오를 검색하거나 업로드 할 수 있는지 여부를 제어 할 수 있습니다.

계정 관리 랜딩 페이지

계정 관리 랜딩 페이지에는 작성자 및 뷰어에 대해 할당 된 그룹과 함께 구성된 계정의 현재 목록이 표시됩니다.

  1. 계정 제목 -현재 구성된 (또는 최소한 생성 된) 계정.
  2. AuthorsGroup -SharePoint 그룹에 계정에 대한 "작성자"권한이 할당되었습니다.
  3. ViewersGroup -SharePoint 그룹에 계정에 대한 "뷰어"권한이 할당되었습니다.
  4. 새 계정 추가 -새 계정을 설정하려면 계정 생성 화면을 엽니 다.

계정 생성 / 편집

새 계정을 만들 든 기존 계정을 편집하든 프로세스와 옵션은 동일합니다. 모든 계정 속성 (작성자 및 뷰어 그룹 제외)은 먼저 Video Cloud 계정에서 가져와야합니다. 다음은 각 계정에 필요한 속성입니다.

  1. 계정 이름 -계정 이름. 이 이름은 SharePoint에서만 사용되므로 Video Cloud와 동일 할 필요는 없습니다.
  2. DefaultVideoPlayerId -동영상의 기본 플레이어를 설정합니다.
  3. DefaultPlaylistPlayerId -기본 재생 목록 플레이어를 설정합니다.
  4. 계정 ID -Video Cloud의 Brightcove 계정 ID.
  5. ClientID -Brightcove API 인증 클라이언트 ID.
  6. 클라이언트 비밀 -Brightcove API 인증 클라이언트 시크릿.
  7. 임시 저장 위치 -동영상 및 텍스트 트랙을 Video Cloud에 업로드 할 때 사용할 임시 저장 위치입니다. 드롭 다운을 사용하면 Amazon Web Services (AWS) 또는 Dropbox를 선택할 수 있습니다.
  8. AWS 액세스 키 ID -AWS를 사용하는 경우 여기에 액세스 키 ID를 입력합니다.
  9. AWS 보안 액세스 키 -AWS를 사용하는 경우 여기에 보안 액세스 키를 입력합니다.
  10. AWS 버킷 이름 -AWS를 사용하는 경우 여기에 AWS 버킷 이름을 입력합니다.
  11. Dropbox 계정 ID -Dropbox를 사용하는 경우 여기에 Dropbox 계정 ID를 입력합니다.
  12. Dropbox 액세스 토큰 -Dropbox를 사용하는 경우 여기에 Dropbox 액세스 토큰을 입력합니다.
  13. 사용자 정의 필드 -이 계정에 대해 Video Cloud에서 생성 된 사용자 정의 필드를 매핑하는 데 사용됩니다. 자세한 내용은사용자 지정 필드 매핑섹션을 참조하십시오.
  14. 저자 그룹, 뷰어 그룹 -계정에 대한 SharePoint 측 권한 정의 그룹.
    1. 저자 그룹의 구성원 (이 경우 SharePoint “소유자” 그룹) 은 비디오 추가, 기존 비디오 편집 및 업데이트, 재생 목록 생성/편집 등과 같은 쓰기 토큰이 필요한 계정에서 기능을 수행 할 수 있습니다. 또한 비디오 및 재생 목록 탐색 및 검색과 같은 모든 읽기 유형 기능을 수행 할 수 있습니다.
    2. 뷰어 그룹 (이 경우 SharePoint "Visitors"그룹)의 구성원은 비디오 및 재생 목록 탐색 및 검색과 같은 읽기 유형 기능에 대해서만 권한이 있습니다.

위 계정 목록의 속성을 편집하려면 사용자가 속성 이름 또는 해당 값 (있는 경우)을 클릭하여 속성 팝업 대화 상자를 열 수 있습니다.

Brightcove의 계정 자격 증명을 SharePoint에 입력

SharePoint를 Brightcove Video Cloud에 연결하려면 Brightcove의 계정 자격 증명을 SharePoint에 추가해야합니다. 이 섹션에서는 Brightcove에서 계정 자격 증명을 찾는 방법을 설명합니다. 다음 섹션에서는 이러한 자격 증명을 SharePoint 커넥터에 추가하는 방법에 대해 설명합니다.

  1. Brightcove Video Cloud에 로그인합니다. 게시자 ID 값을 기록해 둡니다.
  2. 관리자로 이동합니다.
  3. API 인증으로 이동합니다.
  4. 새 응용 프로그램을 등록하십시오.
  5. 애플리케이션 이름을 지정하고 계정 액세스를 설정하고 API를 활성화합니다.
  6. 클라이언트 ID와 클라이언트 시크릿을 기록해 둡니다. 다음 섹션에서 SharePoint에 입력해야 합니다.

Brightcove 계정 ID를 설정하려면 "계정 ID"버튼을 선택하고 이전 단계에서 저장 한 Brightcove 게시자 ID를 다음 모달에 입력합니다.

Brightcove 클라이언트 ID를 설정하려면 "클라이언트 ID"버튼을 선택하고 이전 단계에서 저장 한 클라이언트 ID를 다음 모달에 입력합니다.

Brightcove 클라이언트 시크릿을 설정하려면 "클라이언트 시크릿"버튼을 선택하고 이전 단계에서 저장 한 클라이언트 시크릿을 다음 모달에 입력합니다.

  1. 값 -속성 값을 입력합니다. 사용 가능한 경우 Video Cloud 데이터에서 잘라 내기 및 붙여 넣기를 사용하여 값을 입력 할 수 있습니다.
  2. 닫기 -변경하지 않고 속성 대화 상자를 닫습니다.
  3. 확인 -속성 목록에 값을 기록하고 대화 상자를 닫습니다.

비디오 클라우드 플레이어 정의

SharePoint에서 기본 단일 비디오 플레이어로 사용해야하는 플레이어 ID를 설정하려면 "DefaultVideoPlayerId"단추를 선택하고 다음 모달에서 Brightcove의 플레이어 ID를 입력하십시오.

  1. 값 -속성 값을 입력합니다. 가능한 경우 Video Cloud 데이터에서 잘라 내기 및 붙여 넣기를 사용하여 값을 입력 할 수 있습니다.
  2. 닫기 -변경하지 않고 속성 대화 상자를 닫습니다.
  3. 확인 -속성 목록에 값을 기록하고 대화 상자를 닫습니다.

SharePoint에서 기본 재생 목록 비디오 플레이어로 사용해야하는 플레이어 ID를 설정하려면 "DefaultPlaylistPlayerId"단추를 선택하고 Brightcove의 플레이어 ID를 입력하십시오.

사용자 정의 필드 정의

Brightcove 커넥터에서 계정을 설정할 때 Video Cloud에서 생성 된 사용자 정의 필드를 커넥터의 계정 프로필에 추가하여 비디오 추가 및 비디오 관리 영역에 표시되고 값이 저장되도록 할 수 있습니다. 검색을 위해 SharePoint에서. 사용자 정의 필드가 매핑되면 필수 필드인지 여부와 함께 계정 관리 화면의 사용자 정의 필드 섹션에 나열됩니다.

사용자 정의 필드를 추가, 삭제 또는 편집하려면 "사용자 정의 필드 편집"단추를 선택하십시오. 다음과 같은 모달 창이 표시됩니다. 모두 계정에 사용할 수 있는 사용자 정의 필드 중 참고-이 목록은 새 계정을 처음으로 저장 한 경우에만 사용할 수 있습니다.

  1. Video Cloud 사용자 정의 필드 -선택한 계정에 대해 Video Cloud에 정의 된 모든 사용자 정의 필드의 이름.
  2. 유형 -사용자 정의 필드 유형 (텍스트 또는 목록).
  3. 필수 -사용자 정의 필드가 필요한지 여부를 나타냅니다. 이 정의는 Video Cloud에서 설정되며 커넥터 내에서 변경할 수 없습니다.
  4. 포함 -비디오를 추가, 편집 또는 관리 할 때 표시되도록 커넥터에서 사용자 정의 필드를 계정의 일부로 만듭니다.

임시 업로드 스토리지를 위해 Dropbox 또는 AWS S3에 연결

Brightcove API를 사용하려면 Video Cloud에서 처리 할 모든 파일 (비디오 파일, 텍스트 트랙 파일)을 Brightcove 커넥터와 Video Cloud 모두에서 액세스 할 수있는 중간 위치에 업로드해야합니다. AWS S3 및 Dropbox는 Brightcove 커넥터에서 사용할 수있는 두 가지 권장 서비스입니다.

Brightcove 커넥터의 계정 설정 섹션에 필요한 정보를 생성하기 위해 이러한 서비스를 구성하는 방법에 대한 지침은 다음 링크를 사용하여 찾을 수 있습니다.

  • 개요: Dynamic Ingest API
    S3에서 동적 수집 사용
    http://docs.aws.amazon.com/AmazonS3/latest/gsg/CreatingABucket.html
    https://www.dropbox.com/developers

참고-AWS S3 또는 Dropbox에 업로드 된 파일은 자동으로 삭제되지 않습니다. 일정 시간이 지나면 자동으로 파일을 삭제하도록 이러한 서비스에 대한 정책을 설정하는 것이 좋습니다.

임시 저장 위치 팝업

업로드 파일을 사용해야하는 임시 저장 위치를 설정하려면 "임시 저장 위치"버튼을 선택하고 풀다운에서 저장 옵션 중 하나를 선택합니다.

  1. 값 -임시 저장 위치로 Amazon Web Services 또는 Dropbox를 선택합니다.
  2. 확인 -속성 목록에 값을 기록하고 대화 상자를 닫습니다.

선택한 스토리지 위치에 따라 "AWS 액세스 키 ID", "AWS 보안 액세스 키"및 "AWS 버킷 이름"버튼을 선택하여 AWS S3에 필요한 구성 데이터를 입력하거나 "Dropbox 계정 ID"및 "Dropbox를 선택합니다. Access Token "버튼을 눌러 Dropbox에 필요한 구성 데이터를 입력합니다.

SharePoint 권한 설정

작성자 그룹으로 인증되어야하는 SharePoint 그룹을 설정하려면 "작성자 그룹"버튼을 선택하고 다음 모달에 설명 된대로 풀다운에서 그룹을 선택합니다.

그룹 속성 팝업

  1. 그룹 선택 -커넥터가 설치된 사이트에서 사용 가능한 모든 SharePoint 그룹을 표시합니다.
  2. 확인 -속성 목록에 값을 기록하고 대화 상자를 닫습니다.

뷰어 그룹으로 인증되어야하는 SharePoint 그룹을 설정하려면 "뷰어 그룹"버튼을 선택하고 풀다운에서 그룹을 선택합니다.

중요-계정 값은 계정 "저장"버튼을 클릭 할 때까지 저장되지 않습니다. 속성 대화 상자에 값을 입력해도 계정에 즉시 저장되지는 않습니다.

  1. 계정 저장 -계정 속성에 설정된 모든 값을 SharePoint에 커밋합니다.
  2. 계정 삭제 -SharePoint에서 계정을 제거합니다. 경고-이 작업을 수행하면 해당 계정으로 SharePoint로 가져온 모든 비디오에 더 이상 SharePoint에서 액세스 할 수 없습니다. 그러나 Video Cloud에서 비디오를 삭제하지는 않습니다.
  3. 취소 -변경 사항을 기록하지 않고 계정 설정 창을 닫습니다.

비디오 추가, 가져 오기 및 삭제

비디오 추가

커넥터의 비디오 추가 섹션은 비디오 클라우드에 새 비디오를 업로드하는 데 사용됩니다. 비디오 파일 자체를 제외하고 Video Cloud에 업로드되는 모든 데이터는 SharePoint에도 저장되므로 커넥터 기능과 SharePoint 검색 결과에 사용할 수 있습니다. 또한 SharePoint 관리자와 사용자에게 Video Cloud에 로그인하지 않고도 데이터에 대한 기본 액세스를 제공합니다.

비디오 추가는 단일 비디오와 여러 비디오 (일괄 업로드)를 모두 지원합니다. 사용자가 비디오 추가에 액세스하면 (할 수있는 권한이있는 경우) Video Cloud로 전송 될 메타 데이터를 입력 할 수있는 다음 필드가 표시됩니다.

  2. 이름 (필수)-업로드 된 동영상의 이름입니다.
  3. 간단한 설명 -동영상에 대한 간단한 설명입니다.
  4. 장황한 설명 -동영상에 대한 자세한 설명.
  5. 활성 (필수)-드롭 다운 옵션은 다음과 같습니다. 활성, 비활성, 보류 중, 삭제됨. 참고-동영상이 비활성 상태로 업로드되면 보류 중 또는 삭제됨 SharePoint는 향후 편집을 위해 더 이상 동영상 정보에 액세스 할 수 없습니다. 이것은 Brightcove API의 기능입니다.
  6. 관련 링크 URL, 관련 링크 텍스트 -동영상 플레이어에 표시되는 관련 하이퍼 링크 및 텍스트입니다.
  7. 시작 가능 날짜, 종료 가능 날짜 -비디오 사용 가능 날짜를 정의합니다. 정의 된 경우이 날짜 범위를 벗어나면 동영상이 재생되지 않습니다.

  10. 사용자 정의 필드 -선택한 계정의 커넥터에 추가 된 사용자 정의 필드가 표시됩니다. 사용자 정의 필드에 대해 Video Cloud에 정의 된 모든 설명 텍스트가 필드 아래에 표시됩니다.
  11. 텍스트 트랙 파일 -첫 번째 업로드 또는 후속 업데이트시 업로드 된 각 비디오에 텍스트 트랙을 추가 할 수 있습니다. 텍스트 트랙추가를 참조하십시오.

텍스트 트랙 추가

텍스트 트랙은 첫 번째 업로드 또는 후속 업데이트시 모든 비디오에 추가 할 수 있습니다. 비디오에는 하나 이상의 텍스트 트랙이 연결될 수 있습니다. 사용자는 비디오에 추가 할 텍스트 트랙 유형 (파일 업로드, URL 업로드 또는 원격 URL)을 선택할 수 있습니다. "없음"을 선택하면 섹션에 필드가 표시되지 않습니다.

중대한: 단일 비디오는 파일 업로드, URL 업로드 또는 원격 URL의 혼합을 지원하지 않습니다. 각 비디오에는 한 가지 유형의 텍스트 트랙 만 연결할 수 있습니다.

파일 업로드

텍스트 트랙 파일에 대해 "파일 업로드"를 선택하면 다음 필드가 표시됩니다.

  1. 텍스트 트랙 유형 선택 -파일 업로드, URL 업로드 또는 원격 URL을 선택할 수 있습니다.
  2. 파일 선택기 -텍스트 트랙 파일 선택.
  3. 텍스트 트랙 속성 -텍스트 트랙과 관련된 속성을 설정합니다. "추가"를 클릭하면 다음 업로드 / 업데이트시 비디오 데이터에 텍스트 트랙이 추가됩니다.
    1. 언어 - 텍스트 트랙의 언어에 대한 두 글자 코드 (유효한 BCP 47 언어 태그) 입니다 (예: 영어의 경우 “en”).
    2. 상표 -자막에 사용할 수있는 여러 언어를 나열하는 메뉴와 같이 사용자에게 표시되는 트랙의 레이블입니다.
    3. 종류 -나열된 다섯 가지 지원되는 트랙 유형 중 하나
  4. 추가 / 대기열 텍스트 트랙 목록 -다음 저장시 업로드 대기중인 기존 텍스트 트랙 및 새 텍스트 트랙 목록을 표시합니다. "제거"링크를 클릭하고 비디오를 저장하여 텍스트 트랙을 삭제할 수 있습니다.

비디오 관리

커넥터의 비디오 관리 섹션은 기존 비디오를 찾아보고 관리하는 데 사용됩니다. 여기에는 필요에 따라 기존 비디오의 편집 및 업데이트가 포함됩니다. 검색 기능은 검색 이름, 설명 및 태그 정보로 비디오를 찾는 데 사용됩니다.

    1. 비디오 이름-이름을 클릭하면 데이터를 편집 할 수있는 비디오가 열립니다.
    2. 간단한 설명
    3. 비디오와 관련된 태그 목록 (Video Cloud에 태그로 저장되어 있어도 참조 ID를 나열하지 않음).
    4. "비디오 편집"링크. 이 링크 (또는 이름 클릭)는 현재 사용자에게 권한이있는 경우 편집 할 비디오를 여는 데 사용됩니다.
    5. 비디오 ID: 이것은 Video Cloud 비디오 ID입니다. 이 ID는 동영상을 편집하고 (변경 사항이 없더라도) 업로드 후 처음으로 저장되면 표시됩니다.

비디오 클라우드 전용 비디오를 SharePoint로 가져 오기

커넥터를 통하지 않고 Video Cloud에 직접 업로드 된 비디오가있는 경우 해당 비디오의 데이터를 SharePoint로 "가져 오기"하여 SharePoint에서 관리 할 수 있습니다. 이렇게하려면 먼저 "비디오 클라우드 전용"목록 옵션 (1)을 사용하여 비디오를 찾습니다.

그런 다음 비디오 이름 또는 비디오 편집 링크를 클릭하여 다음 섹션에 설명 된대로 편집 모드에서 비디오를 엽니 다. 편집 모드에서 비디오를 저장하면 데이터도 SharePoint에 저장되고 "SharePoint 전용"섹션에 비디오가 나타납니다.

중요-SharePoint 커넥터는 SharePoint에서 비디오 클라우드로만 데이터를 동기화하고 비디오 클라우드에서는 SharePoint로 동기화하지 않습니다. 즉, Video Cloud에서 SharePoint로 비디오를 가져 오면 SharePoint에서 메타 데이터를 변경해야하며 Video Cloud에서 변경 한 메타 데이터는 SharePoint에서 업데이트되지 않으며 다음에 해당 비디오를 덮어 쓸 때 덮어 씁니다. SharePoint에서 편집했습니다.

계정에서 기존 비디오 편집

데이터 필드를 업데이트하기 위해 기존 비디오를 편집하려면 사용자가 비디오 목록에서 이름 (1) 또는 비디오 편집 링크 (2)를 클릭 할 수 있습니다.

편집을 위해 선택한 비디오에 대한 데이터가 표시된 비디오 편집 화면이 나타납니다.

  1. "비디오 편집"은 일반적으로 비디오 추가가 표시되는 왼쪽 기본 탐색에 표시됩니다. 기존 동영상이기 때문에 다른 모드로만 동일한 페이지 형식을 사용하기 때문입니다. 또한 필요한 경우 사용자가 비디오 관리 섹션으로 쉽게 돌아갈 수 있습니다.
  2. 이 프로세스 중에 편집 / 업데이트 할 수없는 특정 필드가 있습니다.
    • 비디오 클라우드 계정간에 비디오를 전송할 수 없으므로 비디오와 연결된 계정을 변경할 수 없습니다.
    • 참조 ID (Video Cloud 비디오 ID)는 변경할 수 없습니다.
    • SharePoint ID는 변경할 수 없습니다.

Video Cloud에 의해 자동 생성 된 이미지를 제외한 다른 모든 표시 필드는 비디오를 처음 업로드했을 때와 마찬가지로 업데이트 할 수 있습니다. 변경이 이루어지면 사용자는 "저장"버튼을 클릭하여 변경 사항을 적용합니다. 동영상 추가 업로드에서 실행되는 것과 동일한 유효성 검사가 여기에서도 수행됩니다.

동영상 삭제

SharePoint에서 비디오 삭제는 커넥터 외부에서 수행됩니다. SharePoint에서 비디오를 삭제하려면 비디오를 찾아 삭제할 수있는 사이트 콘텐츠-> 비디오로 이동합니다.

중요-SharePoint에 녹화 된 비디오를 삭제해도 Video Cloud에서 비디오가 삭제되지 않고 비디오를 SharePoint에 연결하는 데 사용 된 일부 태그가있는 Video Cloud에 비디오가 남습니다. 나중에 SharePoint로 비디오를 다시 가져 오려면 "bcsp-"로 시작하는 태그를 제거해야합니다.

동영상 검색

비디오 관리에서 사용자는 검색어를 입력하여 선택한 계정에서 검색어와 일치하는 이름, 설명 또는 태그 정보가있는 비디오를 찾을 수 있습니다. 결과 목록이 표시되면 모든 정렬 및 순서 필터를 적용 할 수 있습니다.

  1. 검색 필드 -사용자는 여기에 검색어를 입력하고 돋보기를 클릭 (또는 Return 키를 누름)하여 검색을 실행할 수 있습니다.
  2. SharePoint 만 또는 Video Cloud 만 표시 -비디오를 SharePoint에 녹화 된 비디오로만 필터링하거나 Video Cloud에만 상주하고 SharePoint로 데이터를 가져온 적이없는 비디오로 필터링합니다.
  3. 정렬 / 순서 필터 -동영상을 더 빨리 찾을 수 있도록 나열된 매개 변수별로 결과를 정렬하고 정렬하는 데 사용됩니다.

재생 목록 추가, 가져 오기 및 삭제

커넥터의 재생 목록 관리 섹션은 새 재생 목록을 만들고 편집 및 업데이트를 포함하여 기존 재생 목록을 찾아보고 관리하는 데 사용됩니다. 비디오와 달리 재생 목록 정보는 SharePoint에 저장되지 않고이 섹션에서 계정을 검색 할 때마다 Video Cloud에서 검색됩니다. 이러한 이유로 API를 통한 재생 목록 데이터의 가용성에 내재 된 약간의 지연이 있으며 다음 섹션에서 설명합니다.

재생 목록 랜딩 페이지 관리

재생 목록 관리 랜딩 페이지에는 재생 목록을 찾아 보거나 사용 가능한 계정에서 새 항목을 만드는 데 사용할 수있는 필드 목록이 표시됩니다.

  1. 계정 선택 -이 드롭 다운 목록은 사용자가 액세스 할 수있는 모든 계정으로 채워집니다. 이는 계정 그룹 할당을 통해 활성화됩니다. 사용자는 재생 목록 목록을 보려면 계정을 선택해야합니다.
  2. 재생 목록 표시 열 -이 열에 재생 목록 데이터가 표시됩니다. 결과는 열 제목을 클릭하여 열별로 정렬 할 수 있습니다.
  3. 새 재생 목록 추가 -이 버튼을 클릭하면 선택한 계정에 새 재생 목록이 생성됩니다.

계정을 선택하면 해당 계정의 모든 재생 목록이 표시됩니다.

  1. 계정 선택 -이 드롭 다운 목록은 사용자가 액세스 할 수있는 모든 계정으로 채워집니다. 이는 계정 그룹 할당을 통해 활성화됩니다. 사용자는 재생 목록 목록을 보려면 계정을 선택해야합니다.
  2. 재생 목록 표시 열 -목록은 열 제목을 클릭하여 열별로 정렬 할 수 있습니다. 목록이 길어지면 재생 목록을 찾는 데 도움이 될 수 있습니다.
  3. 재생 목록 목록 - 각 재생 목록 목록에는 다음이 포함됩니다.
    1. 재생 목록 이름-이름을 클릭하면 데이터 편집을위한 재생 목록이 열립니다.
    2. 재생 목록에서 활성화 된 동영상 ID 목록입니다.
    3. 간단한 설명.
    4. 재생 목록 유형 - 수동 또는 스마트 재생 목록 .
    5. 재생 목록의 ID (Video Cloud 재생 목록 ID)입니다.
    6. "재생 목록 편집"링크. 이 링크 (또는 이름 클릭)는 편집 할 재생 목록을 열거 나 세부 정보를 보는 데 사용됩니다.
  4. 새 재생 목록 추가 -이 버튼을 클릭하면 선택한 계정에 새 재생 목록이 생성됩니다.

새 재생 목록 추가

계정에 새 재생 목록을 추가하려면 사용자는 "새 재생 목록 추가"버튼을 클릭합니다. 그러면 재생 목록 추가 양식이 열립니다.

 

이동할 비디오를 선택하고 "비디오 이동"옵션을 활성화하여 위 / 아래 버튼을 사용하여 재생 목록에서 비디오를 위아래로 이동할 수 있습니다. 한 번에 여러 비디오를 이동할 수 있습니다.

  1. 이름 -새 재생 목록의 이름입니다.
  2. 기술 -재생 목록에 대한 간략한 설명입니다.
  3. 유형 - 재생 목록 유형 - 수동 또는 스마트 . 이 예는 수동 재생 목록을 보여줍니다. "재생 목록 편집"섹션의 예는 스마트 재생 목록에 대한 옵션을 보여줍니다.
  4. 사용 가능한 비디오 -수동 재생 목록의 경우 검색 창과 계정에서 사용 가능한 동영상 목록이 표시됩니다. 이를 통해 사용자는 재생 목록에 추가 할 비디오 선택 범위를 좁힐 수 있습니다.
  5. 사용 가능한 비디오 목록 -계정에서 사용 가능한 동영상을 나열하거나 검색어를 기준으로 필터링합니다. 비디오를 클릭하여 하나 이상의 비디오를 선택할 수 있습니다. 선택한 비디오는 녹색으로 강조 표시됩니다. 비디오 선택을 취소하려면 사용자가 비디오를 다시 클릭합니다. 이렇게하면 한 번의 작업으로 여러 동영상을 재생 목록으로 이동할 수 있습니다.
  6. 액션 화살표 -선택한 비디오를 재생 목록 안팎으로 이동합니다.
  7. 이 재생 목록의 비디오 -재생 목록의 현재 동영상이 여기에 표시됩니다.
  8. 저장 -변경 사항을 적용하고 Video Cloud에서 새 재생 목록을 만드는 프로세스를 시작합니다. 참고-재생 목록은 SharePoint에 저장되지 않습니다.

기존 재생 목록 편집

기존 재생 목록 편집은 재생 목록 목록에서 이름 또는 "편집"링크를 클릭하여 시작됩니다. 재생 목록 옵션을 편집 할 수있는 다음보기가 열립니다. 이 예는 스마트 재생 목록에 대한 옵션을 보여줍니다. 필드를 편집 할 때 재생 목록 ID 및 계정을 제외한 모든 필드를 수정할 수 있습니다.

  1. 이름 -재생 목록의 이름입니다.
  2. 기술 -재생 목록에 대한 간략한 설명입니다.
  3. 유형 - 재생 목록 유형 - 수동 또는 스마트 . 이 예에서는 스마트 재생 목록의 옵션을 다룹니다.
  4. 태그 -스마트 재생 목록의 경우이 필드에서 사용자가 재생 목록에 동적으로 포함 할 동영상을 식별하는 데 사용할 기준과 태그를 지정할 수 있습니다.
    • 기준 드롭 다운에는 "하나 이상 포함"또는 "모두 포함"이 포함됩니다.
  5. 주문 -일치하는 동영상의 순서를 설정하는 방법을 사용자가 지정할 수 있습니다. 선택 항목은 다음과 같습니다.
    • 알파벳순
    • 활성화 된 날짜 (최근 먼저)
    • 활성화 날짜 (가장 오래된 것부터)
    • 총 플레이
    • 후행 주 재생
    • 시작일 (오래된 것부터)
    • 시작일 (최근 먼저)
  6. 제한 -선택한 기준에 따라 재생 목록에 포함될 최대 동영상 수를 설정합니다.
  7. 시사 -기준과 일치하는 비디오를 나열하고 "이 재생 목록의 비디오"창에 표시합니다.
  8. 저장 -변경 사항을 적용하고 Video Cloud에서 재생 목록 업데이트 프로세스를 시작합니다. 참고-재생 목록은 SharePoint에 저장되지 않습니다.
  9. 재생 목록 삭제 -이 옵션은 재생 목록을 Video Cloud에서 삭제 된 것으로 표시합니다. 재생 목록은 더 이상 커넥터 재생 목록 목록에 나타나지 않습니다.

SharePoint 사이트에 비디오 추가

SharePoint 커넥터를 사용하면 콘텐츠 작성자가 다양한 방법으로 사이트에 미디어를 삽입 할 수 있습니다. 이 장에서는 사용 가능한 옵션에 대해 설명합니다.

SharePoint On-Premise 용 Brightcove Player 웹 파트

Brightcove Player 웹 파트 샌드 박스 솔루션은 내부 또는 외부 웹 사이트의 사이트 페이지에 배치되는 웹 파트를 생성하고 Brightcove 비디오를 재생합니다. SharePoint Online은 사용자가 사용자 지정 웹 파트를 추가하는 기능을 제한하므로이 비디오 삽입 메커니즘은 온-프레미스 SharePoint 설치에만 사용할 수 있습니다.

또한 웹 파트는 비디오 및 재생 목록을 더 빠르게 찾을 수 있도록 정렬 및 순서 지정을 통한 계정 선택 및 고급 검색을 포함하여 관리 콘솔의 일부 동일한 기능으로 보강되었습니다.

플레이어 웹 파트 옵션을 보려면 SharePoint 페이지가 "편집"모드 (1)에 있어야 웹 파트 편집 옵션 (2)이 표시됩니다. 웹 파트 특정 옵션은 웹 파트 편집기의 "Brightcove 구성"섹션 (3)에 있습니다.

Brightcove 구성 섹션을 확장하면 플레이어 웹 파트 샌드 박스 솔루션의 업데이트 된 기능이 표시됩니다.

웹 파트에서 재생할 비디오 선택

  1. Brightcove 구성 -웹 파트 편집기의이 섹션에서는 웹 파트를 구성하고 재생할 비디오 / 재생 목록을 선택하기위한 매개 변수에 액세스 할 수 있습니다.
  2. 선수 폭, 선수 높이 -플레이어의 크기를 설정합니다.
  3. 플레이어 ID -사용할 플레이어의 ID를 지정합니다. 다중 계정 환경 인 경우 선택한 플레이어가 선택한 비디오와 동일한 계정에있는 것이 중요합니다. 플레이어에 대한 "선택자"가 없으므로 Video Cloud의 ID를 수동으로 입력해야합니다.
  4. 비디오 ID -비디오 ID 필드는 비디오 선택기를 사용하여 채워집니다. 사용자가 필드 오른쪽에있는 줄임표 상자를 클릭하면 선택자 창이 표시됩니다.
  5. 선택기 창
    • 계정 선택 -현재 사용자에게 접근 권한이있는 계정을 표시합니다.
    • 사용 가능한 비디오 -용어를 포함하는 비디오 결과를 필터링합니다. SharePoint 메타 데이터 필드 (정의 된 경우)와 함께 이름, 태그 및 설명 정보가 검색됩니다. 한 번에 하나의 계정에서만 검색 할 수 있습니다. 교차 계정 검색은 지원되지 않습니다.
    • 정렬 기준 및 정렬 기준
    • 비디오 목록 창 -동영상을 클릭하면 재생 대상으로 선택됩니다. 목록은 30 개의 동영상을 표시하는 것으로 제한되어 있으므로 계정을 처음 선택할 때 계정의 일부 동영상이 표시되지 않을 수 있습니다. 검색 기능을 사용하면 원하는 비디오를 찾기 위해 목록을 구체화 할 수 있습니다.
    • OK 버튼 -선택을 커밋하고 적절한 ID 값으로 동영상 ID 필드를 채 웁니다.

플레이어 ID와 비디오 ID가 설정되면 웹 파트가 구성되고 재생할 준비가됩니다!

SharePoint Online 또는 SharePoint 온-프레미스 용 Brightcove Player 솔루션 패키지

Brightcove 플레이어 솔루션 패키지는 Embed Code 대화 상자에 Embed 코드를 복사 / 붙여 넣기하는 페이지에 포함 된 Brightcove 비디오 용 플레이어 용 Embed를 생성합니다. 이 패키지는 SharePoint Online 또는 SharePoint On-premise 모두에 사용할 수 있습니다.

용법

SharePoint 버튼

SharePoint 사이트 페이지를 편집 할 때 "Brightcove"작업 그룹이 리본에 표시됩니다.

Brightcove 비디오 또는 Brightcove 재생 목록 옵션을 선택합니다.

선택 대화 상자

선택 대화 상자를 통해 사용자는 비디오 또는 재생 목록을 선택할 수 있습니다 (리본에서 선택한 옵션에 따라 다름). 이 대화 상자를 사용하면 Video Cloud 비디오 및 재생 목록을 검색하고 결과를 그리드에 표시 할 수 있습니다.

비디오 또는 재생 목록을 선택하면 사용자는 선택적으로 플레이어, 높이 및 너비를 변경할 수 있습니다. "확인"을 클릭하면 brightcove 플레이어 용 스크립트가 생성되어 대화 상자 하단의 텍스트 영역에 배치됩니다.

그런 다음 두 가지 방법으로 이 스크립트를 SharePoint 페이지에 추가할 수 있습니다.

  • 게시 영역으로 직접
  • 스크립트 편집기 웹 파트를 통해 웹 파트 만 허용하는 페이지에서

참고-이 스크립트는 사용자의 클립 보드에 자동으로 복사됩니다.

SharePoint 페이지에 스크립트 추가

참고: 아래 설명 된 절차는 게시 영역에 대한 것입니다. 웹 파트 만 허용하는 페이지의 경우 스크립트 편집기 웹 파트가 페이지에 있으면 절차는 동일합니다.

페이지에 스크립트를 추가하려면 선택 대화 상자에서 생성 된 스크립트를 가져온 후 게시 영역을 클릭합니다. "삽입"탭에서 "삽입 코드"버튼을 찾습니다.

결과 대화 상자에서 플레이어 삽입 스크립트를 붙여 넣습니다. "확인"을 클릭하십시오. 그러면 페이지에 스크립트 편집기가 삽입됩니다.

페이지가 저장되면 해당 스크립트 편집기가 Brightcove 플레이어로 렌더링됩니다.
